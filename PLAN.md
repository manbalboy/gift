# PLAN

## 1. Task breakdown with priority

### 우선순위: 높음 (High)
- [ ] **[Backend] 파일 I/O 시스템 안정성 강화**
  - **변경 파일 후보**: `api/app/services/workspace.py`
  - **영향 범위**: 워크스페이스 내 파일 읽기/쓰기를 수행하는 워크플로우 실행 로직.
  - **구현 내용**: 파일 I/O 시 OS Lock 획득 실패 및 권한 에러에 대한 구조화된 예외 처리 추가. `logging.error` 기반 로깅 도입.
- [ ] **[Backend] OS Lock 경합 단위 테스트 커버리지 확충**
  - **변경 파일 후보**: `api/tests/test_workspace_security.py`
  - **영향 범위**: 백엔드 테스트 파이프라인.
  - **구현 내용**: `workspace.py` 파일 오픈 과정의 OS Lock 경합 상황을 Mocking하는 단위 테스트 작성.
- [ ] **[Frontend] 워크플로우 캔버스 에러 툴팁 UI 결함 수정**
  - **변경 파일 후보**: `web/src/components/` 하위 ReactFlow 노드/툴팁 컴포넌트 및 CSS 파일
  - **영향 범위**: 대시보드 워크플로우 편집기 화면.
  - **구현 내용**: 매우 긴 에러 메시지 렌더링 시 `word-break`, `white-space: pre-wrap`, `max-width` 등 CSS 속성을 적용하여 UI 캔버스 오버플로우를 방지.

### 우선순위: 중 (Medium)
- [ ] **[Backend] 환경 변수 파싱 로깅 강화**
  - **변경 파일 후보**: `api/app/core/config.py`
  - **영향 범위**: 애플리케이션 초기 구동 및 환경 변수 로드 단계.
  - **구현 내용**: 기형적인 환경 변수 주입으로 인한 파싱 실패 및 기본값(Fallback) 반영 로직 구간에 `logging.warning` 또는 `logging.error` 추가.
- [ ] **[Script] 프리뷰 포트 동적 할당 안정성 개선**
  - **변경 파일 후보**: `web/scripts/check-port.mjs`
  - **영향 범위**: 프론트엔드 프리뷰 서버 및 워커 구동 스크립트.
  - **구현 내용**: 포트 전체 고갈 시 즉시 실패 종료하지 않고, 약간의 지연 대기(Sleep) 후 재평가하는 재시도(Retry) 로직 구현.
- [ ] **[Frontend] (추가 기능) 대시보드 내 시스템 경고 위젯 추가**
  - **변경 파일 후보**: `web/src/components/SystemAlertWidget.tsx` (신규), `api/app/api/logs.py` (신규 API 엔드포인트)
  - **영향 범위**: 대시보드 메인 화면 및 백엔드 로그 제공 API.
  - **근거**: 백엔드에서 파일 Lock 경합 및 환경변수 Fallback 같은 내부 시스템 경고가 추가됨에 따라, 시스템 관리자가 대시보드에서 이를 즉시 인지하고 대처할 수 있도록 가시성을 확보하기 위함. 현재 구현된 대시보드의 운영 및 관측 흐름과 자연스럽게 연결되는 기능임.
  - **구현 경계**: 최근 발생한 백엔드 에러 및 시스템 경고 로그를 최대 50개까지만 조회하여 읽기 전용으로 노출. 복잡한 로그 검색이나 필터링 시스템은 구현 범위에서 제외함.

## 2. MVP scope / out-of-scope

### MVP Scope
- `REVIEW.md`에서 식별된 파일 I/O OS Lock 방어 예외 처리 추가 및 모의 단위 테스트 작성.
- 환경 변수 파싱 에러 및 Fallback 발생 시 백엔드 경고 로깅 연동.
- ReactFlow 캔버스 내 긴 에러 메시지로 인한 텍스트 줄바꿈 실패 및 UI 오버플로우 버그 픽스.
- 사용 가능 포트 고갈 시 대기 및 재시도를 수행하도록 포트 체크 스크립트 고도화.
- 추가 기능으로 제안된 시스템 경고 및 상태를 간편하게 확인할 수 있는 대시보드 모니터링 위젯(최대 50건 노출) 구축.

### Out-of-Scope
- Temporal, LangGraph 등 외부 특화 워크플로우 런타임 플랫폼 연동 (내장 엔진 안정화 및 에러 대응을 최우선으로 진행).
- 에이전트 마켓플레이스 고도화 및 복잡한 과금/권한 제어 시스템 개발.
- 외부 대용량 로그 수집 솔루션(Elasticsearch, Datadog 등) 연동 (로컬 파일 및 DB를 이용한 경량 조회로 제한).
- AWS, GCP 등 클라우드 인프라 프로비저닝 작업 (Docker 컨테이너 환경 기반의 로컬 검증으로 한정).

## 3. Completion criteria
- `api/app/services/workspace.py`에서 예외 처리 블록을 통해 Lock 획득 실패 시 크래시 없이 구조화된 `logging.error`가 남아야 한다.
- `api/tests/test_workspace_security.py`가 OS 에러를 성공적으로 Mocking하여 자동화 단위 테스트를 통과해야 한다.
- 파손된 `.env` 정보로 API 서버를 구동했을 때 Fallback 적용 동작과 함께 터미널 및 로그 파일에 경고 로그가 출력되어야 한다.
- 로컬 개발 환경 포트가 전부 점유된 상태에서 `check-port.mjs` 스크립트를 실행했을 때, 무조건 실패하지 않고 일정 횟수의 재시도(Sleep) 루틴이 실행됨을 확인해야 한다.
- 프론트엔드 ReactFlow 툴팁 컴포넌트에 여백이 전혀 없는 매우 긴 문자열을 강제 주입했을 때, 캔버스를 벗어나지 않고 지정된 너비 안에서 텍스트가 줄바꿈되어야 한다.
- 대시보드 진입 시 우측 또는 상단 위젯을 통해 최근 수집된 시스템 경고 및 에러 내역을 UI 카드 형태로 바로 확인할 수 있어야 한다.

## 4. Risks and test strategy

### Risks
- **다중 동시 접근 타이밍 재현의 어려움**: 파일 OS Lock 경합은 아주 짧은 순간에 발생하므로 실제 운영 환경과 동일한 부하에서 테스트가 까다로울 수 있음.
- **포트 재시도 무한 루프**: 재시도 로직 구현 과정에서 최대 시도 횟수 제한 로직에 결함이 있을 경우 스크립트가 무한 대기에 빠져 파이프라인이 멈출 위험성 존재.

### Test strategy
- **단위 테스트 (Unit Test)**: `unittest.mock`을 사용하여 `workspace.py` 파일 열기 시 `PermissionError` 등 OS 시스템 레벨의 예외를 강제로 발생시키고, 의도한 예외 핸들링 코드 블록이 실행되는지 검증한다.
- **통합 테스트 (Integration Test)**: 오류가 포함된 `.env.test` 설정 파일을 주입하여 서버를 시작하고, 예상되는 `logging.warning` 메시지가 정상 호출되는지 검증한다.
- **UI 모의 테스트 (Visual Test)**: 브라우저 개발자 도구를 이용해 에러 툴팁 DOM 텍스트 노드에 공백이 없는 긴 문자열(Lorem Ipsum 1000자 이상)을 강제 주입한 후 오버플로우 발생 여부를 시각적으로 검증한다.
- **스크립트 동작 테스트**: 더미 프로세스로 대상 포트 구간을 강제 점유한 뒤 `check-port.mjs` 스크립트가 타임아웃 및 지정된 재시도 횟수 초과 후, 무한 대기하지 않고 정상적으로 실패 코드를 반환하며 종료되는지 검증한다.

## 5. Design intent and style direction
- **기획 의도**: 워크플로우 실행을 방해하는 시스템 장애 요인(파일 Lock 경합, 파싱 오류, 포트 고갈)을 사전에 방어하고, 관리자가 장애 원인을 명확하게 인지하여 대처할 수 있도록 투명한 시스템 관측 경험을 제공.
- **디자인 풍**: 모던 대시보드형 디자인, 미니멀하고 직관적인 카드형 UI.
- **시각 원칙**:
  - **컬러**: 심각한 에러(Red)와 시스템 경고(Orange) 상태를 명확히 구분하여 인지도를 높이며, 툴팁과 시스템 위젯 배경은 어두운 톤(Dark Gray)을 사용하여 시각적 대비감을 부여하고 눈의 피로를 최소화.
  - **패딩/마진**: 복잡한 에러 텍스트가 숨쉬듯 편안하게 읽힐 수 있도록 툴팁 내부에 최소 12px~16px의 넉넉한 여백 패딩을 구성.
  - **타이포**: 시스템 로그 및 에러 트레이스 메시지 부분은 기술적 식별력을 높이기 위해 고정폭 폰트(Monospace)를 일관성 있게 적용.
- **반응형 원칙**: 모바일 우선 (Mobile-first). 데스크톱 환경에서는 마우스 위치 기반의 레이어 툴팁 및 사이드 패널로 표시하며, 모바일 화면에서는 텍스트 잘림 방지를 위해 화면 하단 고정형(Bottom Sheet) 스타일로 자연스럽게 전환되도록 반응형을 적용.

## 6. Technology ruleset
- **플랫폼 분류**: web, api
- **web**: React 프레임워크 기반 컴포넌트 개발 (Vite 환경, ReactFlow 활용).
- **api**: FastAPI 기반 백엔드 애플리케이션 개발 (Python 환경).
- **포트 사용 규칙**: 실행 가이드 상 포트가 필요할 경우, 개발 및 프리뷰 환경 충돌을 막기 위해 **3000번대(예: 3000~3099) 포트만 사용**하도록 강제.
