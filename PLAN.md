```markdown
# PLAN

## 1. Task breakdown with priority

### [P0] 버그 픽스 및 예외 처리 강화
- **API 예외 처리 보강**: `webhooks.py` 내 잘못된 `workflow_id` 및 페이로드 수신 시 `422 Unprocessable Entity` 반환 로직 추가
- **자원 누수 방지**: 워크플로우 취소 API 호출 시 비동기 제너레이터(SSE) 커넥션 완전 해제 및 `asyncio.CancelledError` 정상 호출 처리 로직 구현
- **DAG Fallback 수정**: `workflow_engine.py`에서 연결이 없는 독립 노드가 병렬 처리 큐로 올바르게 진입하도록 로직 픽스
- **구동 환경 포트 고정**: 개발 및 프리뷰 환경의 구동 스크립트를 정비하여 프론트엔드 UI 포트를 `3100`, API 포트를 `3101`로 명시적 분리 및 고정

### [P1] 보안 및 권한(RBAC) 강화
- **인가 시스템(RBAC) 적용**: `workflows.py`의 Human Gate 관련 API 엔드포인트에 `reviewer`, `admin` 등 Role 기반 접근 인가 로직 추가
- **감사 로그(Audit Log) 도입**: 휴먼 게이트 승인/반려 시 결정 주체(`decided_by`), 시간(`decided_at`), 페이로드(`payload`) 정보를 데이터베이스에 기록하는 로직 및 스키마 추가

### [P1] 프론트엔드 UI/UX 개선 및 고도화
- **인가 에러 핸들링**: Human Gate 접근 시 403 에러 발생 시 콘솔 에러 대신 권한 부족을 안내하는 친화적인 Fallback 모달 렌더링 기능 추가
- **네트워크 복원력 강화**: 클라이언트 SSE 커넥션 상태(연결/단절) 시각화 컴포넌트 추가 및 단절 시 자동 재연결(Exponential Backoff) 로직 구현
- **[고도화 추가] Human Gate 감사 로그 조회 UI**: Human Gate의 과거 승인/반려 이력을 확인할 수 있는 전용 모달 또는 섹션 뷰어 추가
  - **근거**: 백엔드에서 기록되는 Audit Log를 UI에 노출하여 개발 및 QA 조직 내 검수 이력을 투명하게 관리하기 위함
  - **경계**: 특정 워크플로우 실행의 휴먼 게이트 노드에 대한 과거 결정 내역만 Read-Only 형태로 제공하며 편집 기능은 배제함

### [P2] 테스트 스위트 보강
- **백엔드 테스트**: 강제 취소 시 자원 반환, 웹훅 422 에러, 권한(403) 에러 처리를 검증하는 Pytest 단위/통합 테스트 보강
- **프론트엔드 통합 테스트**: `3100` 포트 기반으로 프론트엔드 Human Gate 승인/반려 워크플로우의 전체 흐름을 검증하는 Playwright E2E 테스트 스크립트 작성

## 2. MVP scope / out-of-scope

**MVP Scope**
- `REVIEW.md`에 명시된 기능적 버그(웹훅 422 응답 누락, 커넥션 좀비 프로세스, DAG 큐 진입 오류) 픽스
- Human Gate 접근 통제를 위한 Role 기반 권한 인가 및 결정 이력 저장(Audit Log)
- 프론트엔드 403 Fallback UI 컴포넌트, SSE 연결 상태 시각화 및 자동 재연결 기능
- 앱 구동 시 프론트엔드 포트 `3100`, API 포트 `3101` 할당 및 분리 설정
- 관련 컴포넌트 및 API 예외 상황에 대한 단위, 통합, Playwright E2E 테스트 스위트 구축
- 고도화 플랜으로 추가된 프론트엔드 내 Audit Log Read-Only 뷰어 모달 구현

**Out-of-scope**
- Human Gate 승인 이외 시스템 전반의 Role 계층 구조 및 동적 권한 생성 시스템 구축
- n8n 스타일의 전면적인 시각화 워크플로우 에디터(ReactFlow) 본격 도입 (향후 페이즈로 연기)
- 3000번대(3100, 3101) 이외의 로컬 및 프리뷰 포트 번호 할당 및 사용

## 3. Completion criteria
- 유효하지 않은 웹훅 호출 시 HTTP 422 코드가 정확히 반환되어야 함
- 워크플로우 취소 시 활성화된 SSE 스트림이 메모리 릭 없이 즉시 해제됨을 서버 로그 및 모니터링으로 증명해야 함
- 권한 없는 사용자의 승인/반려 시도 시 API 단에서 403 에러를 응답하고, 브라우저 화면에 친화적 안내 모달이 노출되어야 함
- Human Gate 승인/반려 기록이 DB에 적재되며, UI를 통해 이력을 정상적으로 조회할 수 있어야 함
- 로컬 실행 스크립트 기동 시 UI 접속은 `http://localhost:3100`, API 통신은 `http://localhost:3101`에서만 이루어져야 함
- 작성된 Playwright E2E 테스트가 모든 Human Gate 시나리오를 통과해야 함

## 4. Risks and test strategy

**Risks**
- 비동기 제너레이터(SSE) 취소 이벤트 포착 지연으로 인한 일시적인 자원 반환 실패 우려
- 클라이언트 측 지수적 백오프 재연결이 너무 빈번하게 발생할 경우 서버 API에 트래픽 부담 가중 가능성

**Test Strategy**
- **단위/통합 테스트**: `asyncio.CancelledError` 발생 시나리오를 Mocking하여 백엔드 자원 해제가 올바른 타이밍에 일어나는지 Pytest로 철저히 검증
- **예외 응답 테스트**: 422 및 403 상태 코드가 정상 반환되는지 확인하는 API 통합 테스트 구축
- **E2E 테스트**: Playwright를 이용해 `3100` 포트를 대상으로 실제 사용자 여정을 모방하여 승인, 반려, 권한 부족 등의 시나리오 자동화 테스트 진행

## 5. Design intent and style direction

- **기획 의도**: 워크플로우 실행의 핵심인 휴먼 인더 루프(Human-in-the-loop) 과정을 직관적이고 안정적으로 노출하며, 에러나 연결 단절 시 사용자 불안을 최소화하여 원활한 리뷰 경험을 제공
- **디자인 풍**: 개발 도구 특성에 맞는 대시보드형 디자인. 정보 위계가 명확한 카드형 레이아웃 및 알림 컴포넌트 적용
- **시각 원칙**:
  - **컬러**: 정상 상태(Green), 대기 및 경고(Yellow/Orange), 에러 및 차단(Red)을 활용한 상태 인지적 컬러 시스템 구축
  - **패딩/마진**: 넓은 여백(16px/24px 기준)을 사용해 텍스트와 로그 데이터의 가독성 향상
  - **타이포**: 시스템 메시지 및 로그는 고정폭(Monospace) 폰트, UI 텍스트는 범용적인 Sans-serif 사용하여 명확한 구분 제공
- **반응형 원칙**: 모바일 우선(Mobile First) 기조로 태블릿 및 모바일 기기에서도 승인/반려 모달 및 로그 확인이 무리 없이 가능한 유연한(Fluid) 레이아웃 설계

## 6. Technology ruleset

- **플랫폼 분류**: web, api
- **web**: React 기반(또는 기존 저장소에 세팅된 프론트엔드 스택)을 활용한 UI/UX 구축 및 프론트엔드 상태 관리
- **api**: FastAPI 기반 백엔드 아키텍처 사용, PostgreSQL 모델 추가(Audit Log 용도) 및 비동기 처리 강화를 통한 로직 구현
```
