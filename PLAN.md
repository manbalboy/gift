  - 예외 발생 시 `logging.error`를 사용한 구조화된 로깅 구현.
- **[P0] OS Lock 경합 모의 단위 테스트 추가 (api/tests/test_workspace_security.py)**
  - `workspace.py`의 파일 핸들링 구문에서 Lock 경합이 발생하는 상황을 모킹(Mocking)하여 검증하는 단위 테스트 추가.
- **[P0] 환경 변수 파싱 Fallback 시스템 경고 로깅 연동 (api/app/core/config.py)**

- **[P1] ReactFlow 에러 스니펫 UI 오버플로우 방지 (web/src/...)**

  - **구현 경계:** 워크플로우 캔버스(ReactFlow)의 에러 노출 툴팁 컴포넌트에 한정하여 텍스트 줄바꿈(CSS `word-break` 및 `white-space`) 속성과 최대 너비(`max-width`) 스타일링만 적용하며, API의 데이터 응답 구조 자체는 변경하지 않습니다.

## 2. MVP scope / out-of-scope
- **MVP Scope**
  - 워크플로우 엔진의 파일 I/O 시스템(OS Lock 처리) 및 환경설정 로딩 안정성 확보.
  - 다중 실행 환경에서 발생하는 동적 포트(3100번대) 고갈 엣지 케이스 방어.
  - 워크플로우 캔버스 상의 에러 메시지 툴팁 시각적 오버플로우 결함 수정.
- **Out-of-scope**
  - 새로운 메시지 브로커(Redis/RabbitMQ 등)를 활용한 분산 락 체계 전면 도입.
  - 3000번대(3100~3199) 이외의 새로운 범위의 포트 추가 스캐닝 확장.

## 3. Completion criteria
- `workspace.py` 파일 I/O 경합 발생 시 애플리케이션 크래시가 발생하지 않고 에러 로그가 남는다.
- `test_workspace_security.py`에 작성된 OS Lock 획득 경합 상황 Mocking 단위 테스트가 성공적으로 통과한다.
- `config.py`에서 악의적이거나 기형적인 설정 주입 시 기본값 반영과 함께 시스템 보안 로그가 정상 기록된다.
- `check-port.mjs` 실행 시 3100~3199번 포트가 모두 점유 중일 경우 즉시 실패하지 않고, 정해진 횟수와 간격만큼 대기한 후 최종 평가한다.
- ReactFlow 워크플로우 에러 툴팁에 띄어쓰기 없는 긴 문자열이 입력되어도 지정된 너비를 넘어가지 않고 자연스럽게 줄바꿈 처리된다.

## 4. Risks and test strategy
- **Risks:** 
  - 단위 테스트용 모킹이 실제 OS 간의 미묘한 Lock 경합 타이밍 이슈를 완벽히 모사하지 못할 가능성.
  - `check-port.mjs`의 재시도 대기로 인한 CI 파이프라인 정지 위험 (무한루프 방지 필수).
- **Test Strategy:**
  - **단위/모킹 테스트:** `pytest`를 활용해 특정 파일 오픈 시 `PermissionError` 혹은 Lock 관련 에러를 강제 주입하여 에러 로깅 함수 호출 여부 검증.
  - **스크립트 테스트:** 3100번대 포트를 더미 서버로 고의 점유시킨 뒤 `check-port.mjs`를 실행시켜 일정 횟수까지만 재시도하고 안전하게 실패하는지 확인.
  - **UI 렌더링 검수:** 긴 예외 메시지 페이로드를 워크플로우 상태 응답에 주입하여 브라우저 환경에서 툴팁의 CSS 레이아웃 제어가 정상 동작하는지 육안 검수.

## 5. Design intent and style direction
- **기획 의도:** 백엔드의 파일 경합 및 포트 고갈 같은 시스템적 불안정성을 해소하고, UI 상에서 방대한 시스템 에러 로그를 읽기 쉽게 만들어 안정적인 초장기 워크플로우 관측 경험을 제공합니다.
- **디자인 풍:** 에러를 표현하는 대시보드형 미니멀 스타일 (불필요한 장식 배제, 정보 중심의 플로팅 툴팁).
- **시각 원칙:** 툴팁은 가독성을 위해 고정폭 폰트(Monospace)를 적용하며, 경고를 나타내는 은은한 붉은 계열 색상을 사용합니다. 적절한 내부 패딩(Padding: 8px 12px)을 보장하고 최대 너비를 제한하여 시각적 피로도를 낮춥니다.
- **반응형 원칙:** 모바일 우선 규칙(Mobile First)에 따라 좁은 화면 진입 시 툴팁 너비를 `max-width: 90vw` 수준으로 제약하여 화면을 이탈하는 결함을 방지합니다.

## 6. Technology ruleset
- **플랫폼 분류:** api, web
- **api:** FastAPI 프레임워크 기반 방어 로직 및 pytest 검증.
- **web:** React 프레임워크 기반의 캔버스 UI(ReactFlow) 렌더링 최적화, Node.js 기반 포트 검사 스크립트 로직 처리.
- **포트 실행 규칙:** 실행 가이드 및 로컬 테스트 구동에 포트가 필요한 경우 반드시 3000번대(예: 3100~3199) 포트만 사용하도록 제한합니다.
