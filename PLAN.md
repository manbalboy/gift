# PLAN

## 1. Task breakdown with priority

- **[P0] 보안 유틸리티 모듈 분리 및 리팩토링**
  - `web/src/components/SystemAlertWidget.tsx` 내부에 강하게 결합된 `sanitizeAlertText`, `MASKED_TOKEN` 및 정규식 로직을 공통 유틸리티(`web/src/utils/security.ts` 등)로 분리.
  - 다른 대시보드 위젯에서도 일관된 보안 정책을 재사용할 수 있도록 구조화.
- **[P1] 로그 목록 가상화(Virtualization/Windowing) 도입**
  - 대량의 경고 로그 누적 시 DOM 노드 증가로 인한 렌더링 부하 및 프레임 드랍을 해결하기 위해 윈도잉 기법 적용.
  - 현재 리스트 구조에 맞는 가상화 라이브러리를 추가하여, 화면에 보이는 노드만 렌더링하도록 뷰 리팩토링.
- **[P2] `http/https` 링크 및 주요 키워드 하이라이팅 파서 추가**
  - 로그 텍스트 내 대응 가이드 문서 등의 URL(`http://`, `https://`) 및 티켓 번호를 클릭 가능한 안전한 외부 앵커(`<a>`) 태그로 변환.
  - 기존 마스킹 처리(XSS 방어 및 시크릿 치환)와 충돌하지 않는 순서로 파싱 적용.
- **[P3] 스크롤 스로틀 로직 튜닝 및 `bottomThreshold` 유동적 계산**
  - 찰나의 순간에 폭포수처럼 쏟아지는 대량 알림 상황과, 디스플레이 극단적 확대(300% 이상) 환경을 대비.
  - 브라우저 엔진의 렌더링 계산 오차를 흡수할 수 있도록 고정된 `bottomThreshold`(16px)를 동적 여유값(예: 확대 배율에 따른 가변값)으로 변경 및 Throttle 타이머 오작동 방어.

## 2. MVP scope / out-of-scope

- **MVP Scope**
  - 프론트엔드 대시보드 컴포넌트(`SystemAlertWidget`)의 리스트 렌더링 성능 최적화.
  - 로그 가독성과 편의성 개선을 위한 URL 하이라이팅 기능 추가.
  - 안정적이고 재사용 가능한 유틸리티 구조 개선 및 스크롤 엣지 케이스 해소.
- **Out-of-scope**
  - 새로운 대시보드 위젯 컴포넌트 추가 생성.
  - 백엔드(FastAPI) 로깅 API 서버 로직 변경 (프론트엔드 최적화에 집중).

## 3. Completion criteria

- 개발 서버 실행 시 3100 포트(3000번대 포트)에서 프론트엔드가 정상 구동되어야 함.
- 10,000건 이상의 로그 데이터가 누적되어도 브라우저 멈춤이나 스크롤 렌더링 지연이 없어야 함(가상화 렌더링 확인).
- 로그 메시지에 포함된 URL이 클릭 가능한 외부 링크로 렌더링되며, 클릭 시 새 탭(`target="_blank" rel="noopener noreferrer"`)으로 이동해야 함.
- OS별 휠 가속도 환경과 다양한 화면 배율에서도 자동 스크롤과 PAUSE/LIVE 상태가 정확하게 갱신되어야 함.
- 분리된 공통 보안 유틸리티가 XSS 취약점 없이 기존과 동일하게 시크릿 키를 마스킹하는지 모든 테스트 케이스를 통과해야 함.

## 4. Risks and test strategy

- **Risks**
  - 가상화(Virtualization) 뷰 렌더링 시 기존의 고유 DOM 속성이나 스크롤 훅 로직이 무효화될 수 있어, 스크롤 하단 고정 기능이 오작동할 가능성.
  - 링크 파서와 기존 텍스트 살균 기능이 중첩 시 파싱 에러나 악성 스크립트 우회 가능성.
- **Test Strategy**
  - **단위 테스트:** 분리된 살균 로직(`security.ts`) 및 하이라이트 파서 함수에 대한 입력-출력 검증 (특히 XSS 페이로드 및 시크릿 키 복합 데이터 주입).
  - **E2E 테스트 (Playwright):** 다양한 스크롤 가속도를 모사하여 가상화 리스트 환경 내 스크롤 상태 전이(PAUSE/LIVE) 및 화면 배율(300%) 적용 후 최하단 도달 검증.

## 5. Design intent and style direction

- **기획 의도:** 수백, 수천 개의 시스템 경고가 쏟아지는 긴급 상황에서도 관리자가 병목 없이 알림을 추적하고, 제공된 조치 링크(URL)를 즉각 눌러 트러블슈팅을 이어갈 수 있는 안정적 경험을 제공.
- **디자인 풍:** 기술적이고 데이터 중심적인 대시보드형 (미니멀, 텍스트 중심).
- **시각 원칙:** 
  - 터미널 감성을 유지하기 위해 고정폭(Monospace) 타이포그래피 활용.
  - 링크와 하이라이트는 배경과 대비되는 선명한 블루/형광 계열을 사용하되 시각적 피로도를 낮추도록 마진 조절.
  - 에러와 경고 등 중요한 위험도(Risk) 라벨은 기존 컬러 시스템(Red, Yellow 등)을 엄격히 계승.
- **반응형 원칙:** 
  - 모바일 우선 규칙 적용.
  - 디바이스 환경이 좁더라도 텍스트 Word-wrap을 보장하고, 중요한 버튼 텍스트는 축약어(CLR, EXP)로 전환하여 레이아웃 깨짐을 방지.

## 6. Technology ruleset

- **플랫폼 분류:** web
- **기반 프레임워크/라이브러리:** React 기반 프론트엔드 환경 (Vite 사용). 성능 가상화를 위해 추가적인 React 호환 라이브러리 도입 가능.
- **실행 환경 및 포트:** 개발 구동 시 `PORT=3100 npm run dev`와 같이 3000번대 포트 사용 보장.
