# PLAN

## 1. Task breakdown with priority

**[P0] 보안 및 인가 통제 강화 (Security & Authorization)**
- `web/src/utils/sanitize.ts` 수정: `DOMPurify` 훅(Hook) 기능을 추가하여 `javascript:` 프로토콜 링크를 원천 차단하고, 안전하지 않은 SVG 속성 렌더링을 필터링하는 로직 구현.
- `api/tests/test_workspace_security.py` 작성/수정: 다중 워크스페이스 환경을 모의하여, 인가되지 않은 타 워크스페이스 사용자의 휴먼 게이트 조작 시나리오에서 일관되게 `403 Forbidden`이 반환되는지 검증하는 통합 테스트 추가.

**[P0] 클라이언트 사용성 및 에러 복원력 확보 (Resilience)**
- 프론트엔드 SSE 재연결 로직 수정: Nginx 타임아웃(180s) 강제 종료 후 재연결 시 발생하는 429 에러(Too Many Requests) 방지를 위해, 재시도 간격이 점진적으로 증가하는 Exponential Backoff(지수 백오프) 적용.
- API/Nginx 통신 지연 및 실패(502 Bad Gateway 등) 시 Graceful UI 폴백: 로컬 구동 시 포트 충돌이나 서버 다운으로 3100번대(예: 3108 프록시, 3101 백엔드) 네트워크 연결에 실패할 경우, 무한 로딩이나 화면 깨짐 없이 안전한 상태 복구 및 에러 안내 UI 화면 렌더링.

**[P1] 대용량 산출물 렌더링 최적화 (Performance)**
- `web/src/components/` Artifact 뷰어 수정: 브라우저 힙 메모리 누수(OOM) 방지를 위해, 50MB 이상의 대규모 산출물 텍스트를 한 번에 메모리에 적재하지 않고 스크롤에 맞춰 청크(Chunk) 단위로 가져오고 해제하는 Chunked Loading 구조로 리팩토링.

**[P2] 리뷰 경험 고도화 (Enhancements - REVIEW 연계 인접 기능)**
- **Artifact 뷰어 내장 검색 기능 (Search in Viewer)**: Chunked Loading 도입 시 브라우저 기본 검색(`Ctrl+F`)이 일부 영역만 탐색하는 문제를 보완하기 위해, 뷰어 컴포넌트 자체에 텍스트 검색 및 하이라이팅을 지원하는 툴바 추가. (범위: 전체 시스템이 아닌 단일 Artifact 열람 화면에 국한)
- **휴먼 게이트 거절 사유 템플릿화 (Rejection Templates)**: 권한이 검증된 리뷰어가 휴먼 게이트에서 거절을 선택할 때, 텍스트를 매번 작성하지 않도록 "UX 가이드 위반", "보안 취약점", "사양 불일치" 등 자주 쓰이는 사유를 퀵 프리셋 버튼으로 제공. (범위: 커스텀 기능 제외, 고정 프리셋 3~4종 제공)

## 2. MVP scope / out-of-scope

**MVP Scope**
- `DOMPurify` 훅을 이용한 엄격한 마크다운/SVG 방어 체계 구축
- 격리된 워크스페이스 간 권한 침해를 방어하는 API 보안 통합 테스트 작성
- 서버 타임아웃에 대비한 SSE 클라이언트 지수 백오프 재연결 로직 적용
- API 502/네트워크 타임아웃 에러 상황에 대응하는 프론트엔드 Graceful Fallback UI
- DOM과 메모리를 동시 보호하는 Artifact 뷰어 청크 로딩(Chunked Loading) 로직
- 청크 로딩과 호환되는 뷰어 내부 텍스트 검색 및 하이라이팅 툴바
- 휴먼 게이트 피드백 일관성을 위한 거절 사유 정적 프리셋 버튼 제공

**Out-of-scope**
- 서버사이드 기반의 텍스트 청크 스트리밍 API 신규 구축 (클라이언트 메모리 최적화에 집중)
- 사용자가 직접 추가/수정 가능한 커스텀 거절 사유 템플릿 관리 기능
- 전체 산출물을 가로지르는 글로벌 통합 검색 엔진 구축
- Nginx 설정을 벗어난 인프라 단위의 로드밸런서 타임아웃 구조 변경

## 3. Completion criteria

- `web/src/utils/sanitize.ts`에서 악의적인 스크립트(`javascript:alert(1)`) 및 `<svg>` 페이로드를 입력했을 때 렌더링 전 완전히 제거됨이 단위 테스트로 증명되어야 합니다.
- `pytest` 실행 시 `test_workspace_security.py`의 접근 권한 분리 테스트 케이스가 성공해야 합니다.
- 서버에서 SSE 연결을 인위적으로 끊었을 때, 클라이언트의 네트워크 탭에서 즉시 재시도하지 않고 1초, 2초, 4초, 8초 등으로 대기 시간이 늘어나며 재요청함이 확인되어야 합니다.
- 로컬의 3108번 포트(또는 연관 3100번대 포트) 서비스를 종료했을 때, 브라우저에서 안내 메시지("서버 상태가 불안정합니다")가 담긴 대체 UI가 즉각 렌더링되어야 합니다.
- 크기가 큰 텍스트 산출물 열람 시 브라우저 프로파일러의 메모리 탭에서 Heap 용량이 급격히 증가하거나 스크롤 지연이 발생하지 않아야 합니다.
- 뷰어 상단 검색창에 키워드를 입력하면, 화면상에 일치하는 텍스트 블록이 명확하게 시각적(하이라이트)으로 구분되어야 합니다.
- 휴먼 게이트 거절 모달에서 프리셋 버튼 클릭 시 해당 사유 텍스트가 폼에 즉시 입력되어야 합니다.

## 4. Risks and test strategy

**Risks**
- **렌더링 호환성**: `DOMPurify` 정책이 과도하게 엄격해질 경우, 마크다운에 정상적으로 포함된 유용한 시각적 데이터(예: Mermaid 다이어그램, 안전한 외부 이미지)까지 차단될 우려가 있습니다.
- **스크롤 계산 오류**: VirtualScroll과 Chunked Loading이 겹치면서 스크롤바 높이나 휠 이동 좌표 계산에 오차가 발생해 화면이 튀는 현상이 생길 수 있습니다.

**Test Strategy**
- **보안/단위 테스트**: 다양한 공격 벡터를 포함한 더미 마크다운 데이터를 구성하여, Sanitize 유틸리티가 정상 콘텐츠는 보존하고 위험 요소만 완벽하게 제거하는지 정밀 검증.
- **메모리 프로파일링 테스트**: 10만 줄, 50MB 규모의 텍스트를 프론트엔드 컴포넌트에 주입한 뒤 Chrome DevTools를 활용해 OOM(Out of Memory) 발생 여부 및 GC(Garbage Collection) 추이를 확인.
- **E2E 네트워크 결함 테스트**: Playwright 등을 활용해 서버 강제 종료 시 Graceful UI 전환 및 지수형 백오프 재연결 동작이 사용자에게 자연스럽게 노출되는지 자동화 시나리오 검증.
- **API 통합 테스트**: `asyncio.gather`를 활용한 다중 트랜잭션과 별도로, 워크스페이스 간 권한(Token) 교차 공격 시나리오를 작성하여 403 응답 로직을 보호.

## 5. Design intent and style direction

- **기획 의도**: 예기치 못한 네트워크 장애나 대규모 산출물 앞에서도 멈춤 없이 작동하여, 사용자에게 높은 신뢰감과 끊김 없는 리뷰 워크플로우를 제공하는 것이 핵심입니다.
- **디자인 풍**: '모던 대시보드형(Modern Dashboard)'. 복잡한 데이터를 다루면서도 여백과 구획을 통해 인지적 과부하를 최소화하는 개발자 친화적 디자인을 지향합니다.
- **시각 원칙**:
  - **컬러**: 에러 상태 통보 및 휴먼 게이트 거절 액션은 명확한 경고 색상(Red/Orange)을, 뷰어 내 텍스트 검색 하이라이팅은 눈에 띄는 형광(Yellow) 색상을 적용하여 상태와 액션을 직관적으로 분리합니다.
  - **패딩/마진**: 모듈 간(뷰어, 로그, 툴바) 간격은 `1rem` 이상의 넉넉한 여백을 두어 답답함을 줄이고, 코드 및 로그의 내부 텍스트는 밀도 높게 배치합니다.
  - **타이포**: 시스템 로그, 마크다운 산출물 본문은 가독성을 위해 Monospace 폰트를 강제 적용하고, 제어 버튼과 UI 라벨은 깔끔한 Sans-serif를 사용합니다.
- **반응형 원칙**: '모바일 우선(Mobile First)'. 산출물 열람 및 휴먼 게이트 승인/거절 액션은 모바일 환경에서도 가로 스크롤 이슈나 버튼 터치 영역 겹침 없이 쾌적하게 동작해야 합니다.

## 6. Technology ruleset

- **플랫폼 분류**: `web` 및 `api`
- **Frontend (web)**: React 기반 환경(Vite + React)에서 개발하며, 뷰어 성능 최적화를 위한 Custom Hook 로직과 DOMPurify 처리를 계획합니다.
- **Backend (api)**: FastAPI 프레임워크 기반 하에 Pytest를 활용하여 라우터 권한 격리 및 에러 핸들링 코드를 계획합니다.
- **포트 및 네트워크 규약**: 로컬 프리뷰 및 실행 가이드 적용 시, API 서버와 Nginx 프록시는 **3100번대 포트(예: 3108, 3101 등)** 규약을 준수하여 로컬 충돌을 방지합니다.
