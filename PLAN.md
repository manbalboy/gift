## 2. MVP scope / out-of-scope

- **MVP Scope**:
  - `ErrorLogModal` 클립보드 복사 기능의 Mocking 기반 단위 테스트 구현.
  - 대용량 로그 텍스트 처리용 CSS 점검 및 오버플로우/줄바꿈 규칙 적용.
  - 보안 유틸리티 함수의 비정형 XSS 패턴 우회 방어 테스트 보강.
  - 고도화 플랜의 로그 텍스트 Truncation 기능 및 복사 완료 Toast 알림 기능 구현.
- **Out-of-scope**:
  - `Self-Improvement Loop Engine` 백엔드 핵심 분석/평가 로직의 대규모 리팩토링 및 아키텍처 변경.
  - 리뷰를 통해 안정성이 확보된 `LoopMonitorWidget` 등의 기존 완료 컴포넌트 재설계.
  - 에러 로그 시스템 외의 완전히 새로운 대시보드 화면 추가.

## 3. Completion criteria

- `web/src/components/ErrorLogModal.test.tsx`가 클립보드 복사 성공/실패 Mock 테스트를 모두 통과해야 합니다.
- 수만 자 이상의 에러 텍스트가 렌더링될 때, CSS 및 Truncation 로직을 통해 레이아웃 붕괴나 브라우저 멈춤 현상(렌더링 병목)이 발생하지 않아야 합니다.
- `web/src/utils/security.test.ts`에서 신규 추가된 심화 XSS 페이로드 공격이 성공적으로 차단됨을 테스트 코드로 증명해야 합니다.
- 클라이언트 단의 모든 테스트 스크립트 실행 시 오류 없이 100% 통과해야 합니다.

## 4. Risks and test strategy

- **Risks**:
  - 브라우저 환경에 따라 `navigator.clipboard` API 동작이 상이할 수 있어, Mocking과 실제 런타임 환경 간 불일치 위험이 존재합니다.
  - 정규식을 활용한 XSS 방어 로직 보강 시, 정상적인 제네릭 텍스트(`<T>` 등)가 잘못 필터링되는 회귀 버그(Regression)가 발생할 가능성이 있습니다.
- **Test Strategy**:
  - 브라우저 독립적인 검증을 위해 Jest를 이용한 견고한 Spy 및 Mocking 전략을 수립하여 클립보드 동작을 테스트합니다.
  - 기존에 확보된 엣지 케이스 테스트를 함께 구동하여, 방어 로직 수정 후에도 기존 보안 규칙과 정상 작동 규칙이 상호 충돌하지 않는지 교차 검증합니다.
  - 포트가 필요한 로컬 테스트 실행 시 충돌을 방지하기 위해 3000번대 포트(예: 프론트엔드 빌드 시 3000, 3100)를 우선적으로 활용합니다.

## 5. Design intent and style direction

- **기획 의도**: Self-Improvement Loop 엔진의 잦은 갱신과 시스템 예외 상황(큐 오버플로우 등)을 사용자가 신속하고 명확하게 파악하여 대응할 수 있도록, 안정적이고 직관적인 에러 모니터링 경험을 제공합니다.
- **디자인 풍**: 불필요한 장식을 배제한 개발자 친화적인 대시보드형의 모던 미니멀 스타일.
- **시각 원칙**:
  - **컬러**: 붉은색 계열을 경고 및 에러의 포인트 컬러로 활용하며, 배경과 텍스트는 고대비 모노톤을 유지하여 정보의 가독성을 극대화합니다.
  - **패딩/마진**: 정보가 밀집된 로그 모달 내부에서도 충분한 패딩(Padding)을 확보하여 시각적 답답함을 해소합니다.
  - **타이포**: 로그 텍스트 영역은 Monospace 폰트를 사용하여 코드 및 로그 식별력을 높이고, 일반적인 UI 요소는 가독성 높은 Sans-serif 폰트를 적용합니다.
- **반응형 원칙**: 모바일 우선(Mobile-first) 규칙을 기반으로 하되, 대시보드의 특성을 고려하여 데스크탑 화면에서 넓은 모달과 스크롤이 자연스럽게 연동되는 가변 레이아웃을 유지합니다.

## 6. Technology ruleset

- **플랫폼 분류**: web 및 api
- **프론트엔드 (web)**: React 기반 프레임워크(Vite/TypeScript)를 활용하여 상태 관리 및 컴포넌트 렌더링 성능 최적화를 고려해 계획합니다. 에러 모달 렌더링 시 최적화를 위해 React의 특징을 적극 활용합니다.
- **백엔드 (api)**: FastAPI 기반 엔진과 안정적으로 연동되도록 구성하며, 로컬 환경에서 실행 또는 디버깅 시 3000번대 포트(예: 3100)를 사용해 프론트엔드와 안전하게 통신하도록 설정합니다.
