# PLAN

## 1. Task breakdown with priority
- **P0: 보안 및 코어 신뢰성 확보**
  - 백엔드(api) API의 CORS 허용 도메인(`manbalboy.com`, `localhost`, `127.0.0.1`) 검증 및 설정 반영 (포트 3101).
  - 프론트엔드(web) 대시보드에서 `status.md` 및 아티팩트 내용 렌더링 시 XSS 방지를 위한 DOM 살균(Sanitization) 파이프라인 추가.
  - 백엔드 휴먼 게이트 승인/수정/거절 API에 중복 요청 방지 및 멱등성(Idempotency) 보장 제어 로직 적용.

- **P1: SSE 연결 안정성 강화**
  - 백엔드 API의 SSE 응답에 대한 Keep-Alive 핑(Ping) 주기 설정.
  - 프론트엔드(포트 3100)의 SSE 자동 재연결 및 누락된 이벤트 복구/동기화 로직 구현.
  - Nginx 리버스 프록시 환경(도커) 구성 및 장기 유지보수를 위한 SSE 부하/복원력 시뮬레이션 테스트 시나리오 작성.

- **P2: 대시보드 관측성 고도화 및 상태 관리**
  - 프론트엔드 Audit Log 뷰어에 날짜 범위(`date_range`) 및 승인 상태 필터링 기능 개발.
  - 백엔드 스케줄러를 도입하여 24시간 이상 장기 대기 중인 워크플로우(휴먼 게이트) 탐지 및 알림 이벤트 생성.

- **P3: 테스트 커버리지 확대**
  - 장기 대기 워크플로우 스케줄러 검증을 위한 백엔드 타임 모킹(Time Mocking) 단위 테스트 작성.
  - Playwright를 활용한 프론트엔드 Audit Log 검색, 필터링, 그리고 DOM 렌더링에 대한 E2E 테스트 케이스 추가.

## 2. MVP scope / out-of-scope
**MVP scope:**
- API 서버 및 웹 프론트엔드의 기초적인 보안 통제(엄격한 CORS, XSS 방지 처리) 확립.
- 안정적인 실시간 로그 모니터링을 위한 SSE 스트리밍 아키텍처 개선.
- 동시 요청으로 인한 상태 충돌을 막는 휴먼 게이트 멱등성 로직 및 장기 대기 탐지 기능.
- 대시보드 내 효과적인 로그 및 아티팩트 조회를 위한 필수 필터링 기능(날짜, 상태).
- 모든 기능이 Docker 컨테이너 환경에서 Preview 가능한 상태(포트 7000~7099)로 배포될 수 있도록 패키징.

**Out-of-scope:**
- 완전한 형태의 Visual Workflow Builder(노드 편집기 등)의 신규 UI 구현 (현재는 뷰어 및 승인 관리에 집중).
- 외부 CI/CD 또는 배포 파이프라인과의 신규 이벤트 버스 통합(Idea→Deploy 폐루프 완성은 추후 과제).
- 에이전트 마켓플레이스 화면 구성 및 에이전트 SDK 심화 구현.

## 3. Completion criteria
- **기능 및 보안**: 모든 XSS 취약점 스캔을 통과하고, 허용되지 않은 Origin에서의 API 접근이 차단됨을 확인.
- **안정성**: 로컬 도커 및 Nginx 환경에서 SSE 연결이 1시간 이상 끊김 없이 지속되며, 강제 네트워크 단절 시 프론트엔드가 상태 불일치 없이 재연결됨을 증명.
- **테스트**: 백엔드 단위 테스트(타임 모킹, 멱등성 포함) 및 프론트엔드 Playwright E2E 테스트가 100% 성공(Pass).
- **배포**: PR 생성 시 `http://ssh.manbalboy.com:7000` (또는 7000번대 포트) 대역으로 Docker Preview가 정상적으로 작동하며 접근 가능함.

## 4. Risks and test strategy
- **Risks**: Nginx 등 프록시 환경에서의 장기 연결 타임아웃, 클라이언트 연타로 인한 상태 충돌, 타임존 차이에 따른 검색 결과 누락.
- **Test strategy**:
  - **네트워크 복원력 테스트**: Nginx 컨테이너를 포함한 모의 환경에서 인위적으로 트래픽을 지연시키거나 연결을 끊은 후, 프론트엔드의 이벤트 복원력을 확인.
  - **E2E UI 테스트**: Playwright를 이용해 Audit Log의 날짜 필터링, 상태 변경, DOM 살균 처리된 결과물 렌더링을 자동화하여 검증.
  - **단위 테스트**: 백엔드 스케줄러 로직을 검증하기 위해 시스템 시간을 모킹(Mocking)하여 장기 대기 이벤트를 발생시키고 올바르게 처리하는지 확인.
  - **동시성 테스트**: 휴먼 게이트 API에 동일한 상태 변경 요청을 짧은 시간에 다수 전송하여 멱등성 처리가 정상 작동하는지 확인.

## 5. Design intent and style direction
- **기획 의도**: 개발 파이프라인(DevFlow)의 상태와 로그를 사용자가 즉각적이고 투명하게 인지하여, 개입이 필요한 시점(휴먼 게이트)에 빠르고 정확한 의사결정(승인/거절)을 내릴 수 있도록 돕는 관측 중심의 경험을 제공.
- **디자인 풍**: 군더더기 없는 미니멀하고 모던한 대시보드. 상태 정보가 한눈에 들어오는 카드형 레이아웃 및 타임라인 뷰.
- **시각 원칙**: 코드 블록과 로그 출력부를 위해 모노스페이스(Monospace) 타이포그래피를 적용. 정보의 위계를 명확히 구분할 수 있도록 넉넉한 패딩과 마진을 사용하며, 상태(성공, 실패, 대기)에 따른 직관적인 포인트 컬러 활용.
- **반응형 원칙**: 모바일 우선(Mobile-First) 규칙을 적용하여 스마트폰 기기에서도 진행 상태 확인 및 간단한 승인/거절 조작이 용이하도록 설계. 넓은 화면이 필요한 파일 Diff나 긴 로그 뷰어는 가로 스크롤 및 모달(Modal) 뷰를 활용.

## 6. Technology ruleset
- **플랫폼 분류**: web / api
- **web**: React (Vite 기반) 프레임워크를 사용하여 계획 및 구현. 로컬 개발 시 포트 `3100`을 사용.
- **api**: FastAPI 기반으로 계획 및 구현. 로컬 개발 시 포트 `3101`을 사용.
