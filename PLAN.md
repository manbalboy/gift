# PLAN

## 1. Task breakdown with priority

### 우선순위 P0 (핵심 버그 수정 및 안정성 확보)
- **App 상태 관리 리팩토링 (`web/src/App.tsx`)**
  - `setToasts` 함수 내부에서 렌더링과 동시에 이루어지던 `dedupedToastKeysRef.current.delete()` 등 Ref 조작 로직을 순수 상태 업데이트와 분리.
  - 알림 삭제 이벤트를 처리하는 핸들러 또는 큐 시스템 내부에서 Ref가 안전하고 예측 가능하게 업데이트되도록 생명주기 조정.
- **Toast 리사이즈 성능 최적화 (`web/src/components/Toast.tsx`)**
  - 모바일 뷰포트 및 텍스트 넘침 판단을 위한 `window.addEventListener('resize')` 호출 시 디바운스(Debounce) 혹은 쓰로틀(Throttle) 유틸리티 적용.
  - 컴포넌트 언마운트 시 메모리 누수를 완벽히 방지하기 위한 이벤트 리스너 클린업 로직 보강.

### 우선순위 P1 (UI/UX 개선 및 예외 처리)
- **모바일 Toast 확장 애니메이션 적용 (`web/src/components/Toast.tsx`, `web/src/styles/app.css` 또는 `tokens.css`)**
  - 모바일 환경에서 텍스트 확장(Expand) 탭 시 높이가 갑자기 변하며 화면이 덜컹거리는 현상(Jumping)을 완화하기 위해 CSS `max-height` 및 전환(Transition) 속성 적용.
  - 스택되어 있는 다른 알림 카드들이 자연스럽게 밀려나는 레이아웃 애니메이션 확인.
- **XSS 방어 및 렌더링 무결성 검증**
  - 전체 웹 코드베이스에서 `dangerouslySetInnerHTML` 사용을 원천 배제하는 규칙 점검.
  - 외부에서 주입되는 웹훅 페이로드(`error.detail` 등) 렌더링 시 React의 기본 이스케이프가 안전하게 적용되는지 확인.

### 우선순위 P2 (테스트 커버리지 보강)
- **상태 경합 (Race Condition) 테스트 (`web/src/App.test.tsx`)**
  - 최대 알림 개수(3개) 도달 시, 기존 알림의 수동 닫기(또는 타이머 만료) 시점과 신규 웹훅 알림 수신 시점이 정확히 겹치는 극한의 상황을 모킹하여 중복 알림이 발생하거나 앱 상태가 꼬이지 않는지 단위/통합 테스트 작성.
- **Toast 컴포넌트 분기 단위 테스트 (`web/src/components/Toast.test.tsx`)**
  - 모바일 뷰포트 환경 모킹, 텍스트 말줄임 조건 판단 로직, 확장 버튼 노출 상태에 대한 렌더링 무결성 테스트 추가.
- **E2E 렌더링 점검 (`web/tests/e2e/toast-layering.spec.ts` 등)**
  - 로컬 테스트 서버(포트 `3100` 등 3000번대 사용) 환경에서 동적 브라우저 리사이징 및 모바일 화면 터치 이벤트를 통한 UI 반응성 검증.

## 2. MVP scope / out-of-scope

### MVP Scope (포함 범위)
- 기존 구현된 Toast 알림의 렌더링 부작용(Side Effect) 및 Race Condition 버그 픽스.
- 모바일/데스크톱 리사이즈 이벤트 과부하 방지를 위한 성능 최적화.
- 모바일 텍스트 확장 시 부드러운 전환을 돕는 기초 CSS 트랜지션 처리.
- 핵심 상태 관리 로직 및 리사이즈 예외 처리에 대한 단위 테스트(Jest)와 E2E 테스트(Playwright) 보강.
- 로컬 실행 시 3100번 포트 할당.

### Out-of-scope (제외 범위)
- 백엔드(FastAPI) 로직 확장 및 API 응답 구조의 변경 (프론트엔드 UI/UX 수정에 집중).
- 백엔드 수준에서의 웹훅 페이로드 XSS 사니타이징 (프론트엔드 React 렌더링의 기본 방어 기제로 갈음).
- 알림 보관함(히스토리), 알림별 세부 설정, 데스크톱에서의 드래그 앤 드롭 닫기 등 확장 기능.

## 3. Completion criteria
- `App.tsx`의 상태 생명주기가 React 표준을 준수하며, 콘솔 에러나 렌더링 충돌이 발생하지 않음.
- 연속적인 창 크기 조절 시 디바운스/쓰로틀링이 정상 작동하여 과도한 리렌더링 및 UI 멈춤이 없음.
- 모바일에서 Toast 내용을 확장할 때 주변 레이아웃 깨짐 없이 부드러운 트랜지션이 적용됨.
- 신규 작성된 상태 경합 예방 테스트 및 리사이즈 렌더링 테스트가 CI/로컬에서 모두 100% 통과됨.
- 코드베이스 상에 보안 취약점을 유발하는 `dangerouslySetInnerHTML` 코드가 전혀 존재하지 않음.

## 4. Risks and test strategy

- **동시성 상태 경합 위험**: 사용자의 매우 빠른 연타 혹은 동일한 dedupeKey를 가진 웹훅의 밀리초 단위 동시 수신 시 상태 불일치 발생.
  - **테스트 전략**: `jest.useFakeTimers()`를 적극 활용하여 시간차 이벤트를 세밀하게 조정하고, 동시 이벤트 발생 시 Ref 데이터 세트와 State 배열이 정확히 동기화되는지 단언(Assert).
- **이벤트 기반 렌더링 테스트 한계**: JSDOM 기반 단위 테스트에서는 브라우저의 실제 `resize` 이벤트나 CSS 레이아웃 변경 수치를 완벽히 계산해내기 어려움.
  - **테스트 전략**: 로컬 브라우저 엔진을 사용하는 Playwright E2E 테스트와 모바일 에뮬레이션을 결합하여 실제 창 크기 조절에 따른 텍스트 숨김/노출을 시각적(동작적)으로 교차 검증.

## 5. Design intent and style direction

- **기획 의도**: 시스템 백그라운드의 비동기적 변화(실행 상태, 에러 발생 등)를 사용자에게 즉각적이고 투명하게 전달하되, 현재 수행 중인 워크플로우 편집 등의 메인 작업을 방해하지 않는 쾌적한 피드백 루프 제공.
- **디자인 풍**: 모던, 대시보드형 팝업 스타일. 장식을 배제하고 가독성과 정보의 층위에 집중하는 미니멀 접근 방식.
- **시각 원칙**:
  - **컬러**: 에러/경고/성공에 맞춰 사전에 정의된 상태 색상(디자인 토큰)을 활용.
  - **패딩/마진**: 컴팩트한 여백을 유지하고, 캔버스나 좌측 내비게이션 등 주요 UI를 침범하지 않는 플로팅 마진 적용.
  - **타이포그래피**: 말줄임 처리를 통해 간결함을 우선 유지하고, 클릭/탭 시에만 상세 정보를 보여주는 점진적 노출(Progressive Disclosure) 기법 적용.
- **반응형 원칙**: 모바일 우선(Mobile First). 모바일 기기에서는 화면 너비 제약에 맞춰 카드의 폭을 조정하고 터치 친화적인 확장 영역(최소 44px 이상의 터치 타겟)을 제공.

## 6. Technology ruleset

- **플랫폼 분류**: web (및 지원 API)
- **프론트엔드**: React 기반 SPA 구성, Vite 번들러 사용, 순수 CSS/토큰 기반 스타일링 (Tailwind 배제 원칙 준수).
- **백엔드**: FastAPI (현재 워크플로우 엔진 기반 유지, 본 작업에서는 API 수정 최소화).
- **테스트 스택**: Jest + React Testing Library (단위 테스트), Playwright (E2E 테스트).
- **실행 및 검증 포트**: 충돌 방지를 위해 로컬 개발 및 E2E 테스트 포트는 `3100`번을 타겟으로 실행.
