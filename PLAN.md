# PLAN

## 1. Task breakdown with priority
- **P0: Workflow Engine v2 핵심 실행기 연동 및 고도화**
  - FastAPI 기반의 오케스트레이터 및 실행 엔진 전환 (workflow_id 기반 실행)
  - 노드별 상태(node_runs) DB 저장 및 장애 시 노드 단위 재시도(Retry Policy) 로직 구현
- **P0: Human Gate (승인/반려/수정) 흐름 제어 및 폼 UI 개선**
  - 테스트 및 리뷰 단계 노드에서 상태를 `approval_pending`으로 전환하고 Resume 가능한 API 구현
  - [버그 수정] 거절 사유 폼 프리셋 버튼 연속 클릭 시 띄어쓰기와 개행이 올바르게 텍스트에 병합(Append)되도록 UI 로직 보완
- **P1: 대용량 아티팩트 지원(Artifact-first Workspace) 및 뷰어 최적화**
  - Postgres 및 스토리지 기반 아티팩트 저장 및 검색 아키텍처 연동
  - [고도화 반영 1] SafeArtifactViewer 대용량 렌더링 성능 최적화
    - **근거**: 50MB 이상의 산출물 렌더링 시 발생하는 메모리 부족(OOM) 및 텍스트 청크 교체에 따른 스크롤 좌표 오차를 해결하여 쾌적한 뷰어 경험을 제공하기 위함.
    - **구현 경계**: 가상화 스크롤러의 좌표 보정 로직 개선, 하이라이팅 깜빡임 방지, 50MB 이상 극단적 더미 데이터를 활용한 E2E 메모리 부하 테스트 스크립트 작성에 한정함.
- **P1: Visual Workflow Builder (ReactFlow) 연동 및 검증 강화**
  - ReactFlow를 활용한 워크플로우 시각적 편집기 통합 및 시뮬레이션 런 연결
  - 순환 참조 외에 단절된 노드(Disconnected Graph) 및 다중 Entry 등 비정상 엣지 케이스에 대한 서버측 검증 보완
- **P2: Preview 인프라 보안 강화 및 SSE 스트리밍 안정화**
  - [고도화 반영 2] Preview 환경 접근 권한(Auth) 제어
    - **근거**: 자동 생성된 컨테이너 결과물이 퍼블릭 환경(포트 7000-7099)에 무단 노출되어 코드가 유출되는 보안 리스크를 방어하기 위함.
    - **구현 경계**: Preview 진입 시 일회성 뷰어 토큰 발급 또는 인증 미들웨어 레이어 도입으로 무단 접근을 차단함.
  - Nginx 지연 응답 결함 모킹 테스트 추가 및 SSE 스트리밍 장기 유지 시 발생하는 백엔드/워커 메모리 누수 대응.

## 2. MVP scope / out-of-scope
- **MVP Scope**
  - `workflow_id` 기반의 Workflow Engine v2 코어 흐름 및 노드 상태 추적
  - Visual Builder를 통한 시각적 워크플로우 정의 및 유효성 검증 체계
  - Human Gate를 통한 프로세스 일시 중지 및 수동 개입(Resume) 파이프라인 완성
  - 아티팩트 중심의 기록 뷰어 (수십 MB 단위 데이터 스크롤 대응)
  - 로컬 테스트 환경은 3000번대 포트 활용, 배포 Preview는 7000-7099 대역 포트 개방과 제한된 CORS 정책 수립
- **Out-of-scope**
  - 엔터프라이즈급 역할 기반 접근 제어(RBAC) 및 복잡한 워크스페이스 권한 분리
  - 수천 개의 Job을 동시 처리하기 위한 외부 분산 큐(Temporal 등)의 전면 도입 (MVP는 로컬/내장 엔진 중심 작동 유지)
  - 코어 기능에서 벗어난 외부 CI/CD 플랫폼으로의 범용 확장 플러그인 생태계 구축 (기본 이벤트 트리거만 수용)

## 3. Completion criteria
- FastAPI 오케스트레이터가 Workflow Definition을 정상 파싱하여 엔트리부터 순차적으로 실행하고, 결과를 `node_runs` DB에 성공적으로 적재해야 함.
- Human Gate 이벤트 발생 시 워크플로우 실행이 일시 정지되며, 화면을 통한 사용자의 승인/반려 조작 시 즉각적으로 해당 노드부터 플로우가 재개되어야 함.
- 거절 폼 프리셋 텍스트 입력 버그가 해결되어 정상적인 입력 UX를 제공해야 함.
- 50MB 이상의 극단적인 대용량 아티팩트를 Viewer에 렌더링하는 E2E 메모리 테스트를 통과해야 하며, 스크롤링 시 좌표가 튀는 현상이 없어야 함.
- ReactFlow 시각화 빌더에서 단절되거나 잘못 연결된 워크플로우를 저장 시 API가 명확한 에러와 함께 저장을 거부해야 함.
- 외부 Preview 환경 접속 시 미인가 접근 시도가 성공적으로 차단됨을 테스트로 증명해야 함.

## 4. Risks and test strategy
- **Risks**
  - 프론트엔드 대용량 데이터 로드에 따른 브라우저 크래시 및 UI 블로킹 위험
  - 서버 SSE 커넥션 유실 및 백오프 재연결 과정에서의 엣지 케이스 발생 확률
  - 복잡한 DAG 엣지 케이스를 다루면서 검증 로직에 결함이 발생할 가능성
- **Test Strategy**
  - 대용량(50MB 이상) 더미 데이터를 활용한 렌더링 벤치마크 E2E 스크립트 도입 및 힙 메모리 모니터링 수행
  - 네트워크 단절 및 반쪽 연결을 모킹(Nginx 지연 모킹)하여 SSE 커넥션 누수 방지 통합 테스트 수행
  - Visual Builder 저장 시나리오에 다중 진입점, 격리된 노드 생성 등 다양한 엣지 케이스 단위 테스트(Unit Test) 작성

## 5. Design intent and style direction
- **기획 의도**: 개발/배포 파이프라인의 복잡한 과정을 명확한 노드/엣지 형태로 가시화하고, 오류 복구나 휴먼 컨펌 대기 상황에서도 투명하게 맥락을 파악하여 신속한 의사결정을 돕는 도구 경험 제공.
- **디자인 풍**: 개발자 친화적인 대시보드형 모던 디자인 (어두운 테마 기반의 코드 및 로그 친화적 미니멀리즘)
- **시각 원칙**: 
  - 컬러: 노드 상태(실행 중/성공/실패/승인 대기)를 명확한 시스템 컬러 규칙으로 엄격하게 구별
  - 패딩/마진: 방대한 로그와 코드를 한눈에 많이 담기 위한 컴팩트한 그리드 간격 유지
  - 타이포: 시스템 전반은 가독성 높은 산세리프 폰트를 적용하고, 코드 블럭 및 아티팩트 결과물은 고정폭(Monospace) 폰트를 활용해 정보 전달력 극대화
- **반응형 원칙**: 모바일 우선 규칙 (기본적으로 로그나 뷰어 레이아웃이 모바일 환경에서도 가로 스크롤을 활용하거나 적절히 줄바꿈되어 끊김 없이 정보를 탐색할 수 있도록 유동적 대응)

## 6. Technology ruleset
- **플랫폼 분류**: web, api
- **web**: React 및 Vite 기반 프레임워크로 계획 (ReactFlow를 활용한 워크플로우 캔버스 구성, 가상 스크롤러 적용)
- **api**: FastAPI 기반으로 계획 (Workflow Runner v2, HTTP/SSE 스트리밍, 승인 API 및 외부 트리거 처리)
- **실행 포트**: 로컬 개발 및 실행 환경의 기본 포트는 3000번대를 사용하며, 작업물 Preview 노출은 7000-7099 범위를 준수함.
