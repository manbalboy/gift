## 2. MVP scope / out-of-scope

- **MVP Scope**
  - 브라우저에 적재된 데이터를 바탕으로 Error / Warning 항목만 골라보는 Client-side UI 기능.
  - 과거 로그 탐색 시 스크롤이 튕기지 않도록 방어하는 자동 스크롤 상태(Pause/Resume) 제어 로직.
  - 협소한 모바일 디바이스(320px) 대응을 위한 UI 레이아웃 유연성 확보.
  - 신규 UI 상호작용 및 레이아웃을 검증하는 Playwright E2E 테스트.
- **Out-of-scope**
  - 알림 데이터를 서버 단에서 필터링하여 페이징하는 Server-side 필터 기능 (성능보다 사용자 응답성 향상을 위해 클라이언트 렌더링으로 우선 처리).
  - 새로운 경고 레벨(예: Info, Debug) 파싱 및 신규 UI 요소 확장.
  - Cursor 기반 페이징 방식을 무한 스크롤(Load More) 이외의 완전한 Virtualized List 로직으로 전면 교체하는 아키텍처 변경 작업.

## 3. Completion criteria

- `SystemAlertWidget.tsx` 툴바 영역에 [All], [Error], [Warning] 상태의 필터 칩이 정상 노출되며 클릭 시 화면 상의 목록이 즉각 필터링됨.
- 로그 리스트 영역을 위로 스크롤한 상태에서 시스템 알림이 추가로 수신되어도 사용자의 화면(Viewport) 위치가 최하단으로 강제 이동하지 않음.
- 사용자가 스크롤을 다시 맨 밑으로 내리면 자동 스크롤 추적 기능이 정상적으로 복구되어 새 알림을 즉각 노출함.
- 브라우저 가로 해상도를 320px로 설정했을 때, 새롭게 추가된 필터 칩 영역과 기존의 [CLR], [EXP] 버튼이 서로 겹치거나 가로 스크롤바(오버플로우)가 생성되지 않음.
- 추가된 Playwright E2E 시나리오(로컬 테스트 서버 실행 시 3000번대 포트 사용)가 파이프라인 상에서 충돌 없이 모두 `pass`됨.

## 4. Risks and test strategy

- **Risks**
  - 기존 Cursor 기반 페이징 시스템과 클라이언트 단 필터가 결합될 때, 현재 메모리에 `hasMore` 속성이 남아있더라도 필터 적용 결과가 0개면 사용자에게 '조회할 로그가 더 이상 없다'는 잘못된 인식을 줄 수 있음.
  - 스크롤 이벤트의 과도한 트리거로 인한 React 렌더링 병목 및 성능 하락.
- **Test Strategy**
  - 단위/컴포넌트 테스트: React Testing Library를 활용하여 필터 칩 클릭 시 상태 변경 및 이에 따른 DOM 요소 렌더링 갯수를 단언.
  - E2E 테스트:
    - 모바일 사이즈(320px) 에뮬레이터 환경에서 뷰포트 오버플로우 발생 여부를 시각적 또는 DOM 바운더리 검증을 통해 확인.
    - 대량의 로그를 Mocking하여 발생시키고, 특정 높이만큼 스크롤을 위로 올린 뒤 새로운 데이터가 추가되었을 때 리스트 컨테이너의 `scrollTop` 위치가 변동 없이 고정되어 있는지 검증.

## 5. Design intent and style direction

- **기획 의도**: 실시간으로 쏟아지는 시스템 알림 속에서 치명적인 오류(Error)를 신속하게 식별할 수 있도록 돕고, 과거 로그를 탐색할 때 뷰포트가 강제로 이동하여 시야를 방해받는 경험(UX 결함)을 원천 차단함.
- **디자인 풍**: 기존 위젯의 톤앤매너를 유지하는 모던하고 미니멀한 대시보드 형태.
- **시각 원칙**: Error와 Warning을 직관적으로 구분할 수 있는 컬러 코딩(Red/Orange 계열)을 필터 칩(Chip) 컴포넌트에 적용. 정보의 밀도를 높이면서도 가독성을 해치지 않도록 적절한 여백(Padding/Margin) 배분.
- **반응형 원칙**: 모바일 우선(Mobile-first). 극단적으로 좁은 320px 뷰포트에서도 기능과 조작성을 잃지 않도록 Flex `wrap` 속성 및 가변적인 여백을 사용하여 터치 타겟이 충분히 확보된 상태로 UI 요소들을 배치.

## 6. Technology ruleset

- **플랫폼 분류**: web
- **기반 프레임워크**: React (현재 구현된 `.tsx` 컴포넌트 생태계 확장)
- **스타일링**: 기존 위젯의 CSS 클래스 네이밍 규칙 및 SCSS/CSS 구조 상속 적용
- **테스트**: Playwright를 이용한 종단 간 통합 동작 검증
