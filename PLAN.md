# PLAN

## 1. Task breakdown with priority

### P0 (Critical - 보안 및 시스템 안정성)
- `api/app/main.py`: 서버 CORS 설정을 `manbalboy.com` 및 `localhost` 기반 오리진으로 엄격히 제한.
- `api/app/main.py` (또는 실행 셸 스크립트): 로컬 환경 실행 시 3100번대 포트 충돌 방지 및 재시도/우아한 예외 처리 로직 적용.
- `api/app/services/loop_simulator.py`: `max_loop_count` 및 `budget_limit` 초과 시 루프 엔진을 `stopped` 상태로 전환하는 무한 루프 제어 로직 구현.
- `web/src/utils/security.ts`: `sanitizeAlertText` 함수 반환값에 TypeScript Branded Type 적용 및 JSDoc 문서화를 통해 타입 시스템 수준의 XSS 방어 안정성 강화.

### P1 (High - 기능 결함 수정 및 테스트 보강)
- `web/src/components/ErrorLogModal.tsx`: 대용량 로그 렌더링 시 브라우저 프리징 및 Grapheme Cluster Break 현상을 방지하기 위해 텍스트 분할 로직을 `Intl.Segmenter` 기반으로 변경하고 구형 브라우저 Fallback 처리 추가.
- `api/tests/test_loop_simulator.py`: 예산 한도 소진 및 최대 반복 사이클 도달 시 엔진이 `stopped` 상태로 전이되며 조기 종료되는지 검증하는 단위 테스트 추가.
- `web/src/components/ErrorLogModal.test.tsx`: 10만 자 이상의 텍스트 및 이모지(ZWJ 결합)가 혼합된 대용량 데이터를 주입하여 컴포넌트의 렌더링 무결성을 검증하는 스트레스 테스트 추가.

### P2 (Medium - 엣지 케이스 및 부가 처리)
- 루프 엔진 초기 구동 시 `max_loop_count`가 0 이하로 들어오거나 `budget_limit`이 이미 소진된 상태일 경우 바로 조기 종료(Early exit) 처리.
- 상태가 `stopped`로 전이될 때, 현재 진행 중인 하위 작업(Task)들에 대한 처리(보존 혹은 즉시 취소) 기준 정립 및 예외 처리 구체화.

## 2. MVP scope / out-of-scope

### MVP Scope
- Self-Improvement Loop Engine의 기본적인 실행과 중단 제어(Start, Pause, Resume, Stop).
- 루프 엔진의 폭주를 방어하기 위한 최소한의 자원 제한 장치(`max_loop_count`, `budget_limit`) 연동.
- 대용량 에러 로그 및 복합 문자열의 안전하고 멈춤 없는 프론트엔드 UI 렌더링.
- 보안 요건 충족: 지정된 도메인(localhost, manbalboy)에 대한 CORS 화이트리스트 적용 및 출력 텍스트의 타입 안정성.
- 변경된 사이클 결과물이 Docker 컨테이너에서 문제없이 동작(3100번대 포트 바인딩)하도록 보장.

### Out-of-scope
- 실시간 Analyzer 및 Evaluator 엔진을 위한 별도의 머신러닝 분석/AI 모델 연동 로직 고도화.
- 다중 노드 및 오케스트레이션(Kubernetes 등) 기반의 분산 루프 엔진 스케일링.
- 웹 기반의 전체 코드 변경 사항 병합 도구(Merge Tool) 구축.

## 3. Completion criteria

- REVIEW.md에 명시된 모든 TODO 항목과 결함 사항이 소스 코드 및 설정에 반영되어야 함.
- API 단위 테스트(`pytest`)에서 엔진의 무한 루프 방지 로직 및 초기 한도 초과 등 다양한 엣지 케이스가 전부 PASS될 것.
- 프론트엔드 렌더링 테스트(`Jest`)에서 대용량 텍스트/이모지 삽입 시 깨짐 및 프리징 현상 없이 화면 표출이 확인될 것.
- 허용되지 않은 외부 도메인에서 API 호출 시 정상적으로 CORS 차단이 작동함.
- 로컬 실행 시 3100번대 포트가 이미 점유 중일 경우 애플리케이션이 우아하게 죽거나 재시도하여 포트 바인딩 예외를 처리할 것.

## 4. Risks and test strategy

- **서버 자원 고갈 리스크:** 엔진 루프 제어 실패 시 자원 고갈 우려.
  - **테스트 전략:** `test_loop_simulator.py` 내에서 모의(Mock) 데이터와 한도값을 다양하게 주입하여, 예산 부족, 한계 카운트 도달, 비정상적인 음수 파라미터 유입 시 `stopped` 상태로 정확하게 분기하는지 검증.
- **클라이언트 브라우저 렌더링 병목 리스크:** 단일 컴포넌트 내 무거운 문자열 처리 연산.
  - **테스트 전략:** `ErrorLogModal.test.tsx`에서 대용량의 ZWJ 이모지 및 특수 문자가 결합된 10만 자 문자열을 주입하는 렌더링 벤치마크 테스트 작성.
- **로컬 포트 충돌 리스크:** 동일 머신 내 컨테이너 재실행으로 인한 3100번대 포트 경합.
  - **테스트 전략:** 통합 테스트 혹은 스크립트 실행 레벨에서, 사전 점유된 포트(예: 3100)로 API 구동 시 서버 패닉 대신 안전하게 종료 코드를 반환하는지 검증.

## 5. Design intent and style direction

- **기획 의도:** 지속적으로 코드를 개선하는 자율 엔진(Autonomous Developer)의 안정적인 실행 상태와 에러 기록을, 사용자가 직관적이고 끊김 없이 통제 및 인지할 수 있는 신뢰성 높은 경험을 제공한다.
- **디자인 풍:** 데이터를 명확히 분리하여 보여주는 **모던 대시보드형(Modern Dashboard)**. 군더더기 없는 미니멀리즘과 실용성에 초점.
- **시각 원칙:**
  - **컬러:** 전반적으로 무채색(차콜, 라이트 그레이, 화이트)을 배경으로 사용하며, 엔진의 상태(실행-그린, 중지-레드, 대기-오렌지) 및 시스템 알럿에만 명확한 포인트 컬러를 활용하여 인지적 피드백을 강화.
  - **패딩/마진:** 컴포넌트, 특히 로그 모달과 섹션 간 여백은 넉넉하게(16px~24px 모듈러 스케일) 확보하여 정보 밀도를 낮추고 텍스트 가독성을 최적화.
  - **타이포:** 로깅 및 코드가 출력되는 영역은 고정폭 폰트(Monospace)를 적용하고, 일반적인 조작 UI 및 요약 문구는 시스템 기본 산세리프를 적용해 정보의 성격을 시각적으로 구분.
- **반응형 원칙:** **모바일 우선(Mobile-First)** 규칙 적용. 화면이 좁은 모바일에서는 대화상자/모달을 Full-sheet 형태로 확장하여 공간 효율을 극대화하고, 넓은 데스크탑 환경에서는 화면 중앙의 카드 뷰 혹은 우측 사이드 패널로 고정.

## 6. Technology ruleset

- **플랫폼 분류:** web 및 api
- **web:** React 프레임워크 기반 라이브러리/도구 활용 (함수형 컴포넌트, TypeScript 적용).
- **api:** FastAPI 기반 프레임워크 구현 (의존성 주입 및 Pydantic을 활용한 타입 체크 도입).

## 7. 고도화 플랜 (TODO 확장 반영)

*REVIEW.md에 정의된 TODO를 최우선으로 반영하며, 해당 엔진 제어 및 모니터링 기능과 자연스럽게 연결되는 시스템 안정성 강화 목적의 인접 기능.*

1. **엔진 상태 모니터링 요약 대시보드 위젯 (web)**
   - **근거:** 루프 시뮬레이터가 백그라운드에서 동작할 때, 사용자가 현재의 루프 카운트(Iterated), 소진된 예산(Budget), 그리고 `stopped` 등 엔진의 실시간 진행도를 직관적으로 파악하기 어렵습니다.
   - **구현 경계:** 복잡한 차트나 별도의 SSE 통신 뷰어 없이, API로부터 주기적(Polling)으로 엔진의 요약 상태 데이터만 가져와 화면 상단에 텍스트 형태로 노출하는 가벼운 상태 표시기(React 컴포넌트)로 한정하여 추가합니다.
2. **에러 및 로그 데이터 내보내기(Export) 기능 (web)**
   - **근거:** `ErrorLogModal`의 텍스트 렌더링 안정성을 확보하더라도, 루프 엔진의 주기가 길어질 경우 방대한 로깅 데이터를 외부 환경에서 검토하거나 로컬 파일로 저장해야 할 필요성이 생깁니다.
   - **구현 경계:** `ErrorLogModal`의 헤더 영역에 단순한 "다운로드(TXT/JSON)" 버튼을 추가하고, 백엔드의 파일 생성 로직 없이 클라이언트 사이드에서 현재 불러온 배열 텍스트를 Blob으로 변환하여 즉시 파일 다운로드 처리하는 기능으로 범위를 최소화합니다.
