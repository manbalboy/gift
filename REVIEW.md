# REVIEW

## Functional bugs
- **워크플로우 캔버스 에러 툴팁 UI 결함**: 대시보드 워크플로우 편집기 화면의 ReactFlow 캔버스에서 공백이 없는 매우 긴 에러 메시지가 렌더링될 경우, 텍스트가 줄바꿈되지 않고 툴팁 영역을 벗어나 UI 캔버스 오버플로우를 발생시키는 버그가 존재합니다. (`web/src/components/` 하위 컴포넌트)
- **프리뷰 포트 동적 할당 실패**: 프리뷰 서버 및 워커 구동 시 할당할 수 있는 포트가 전부 점유되어 있다면, 대기나 재시도 로직 없이 `web/scripts/check-port.mjs` 스크립트가 즉시 실패하며 종료됩니다. (재현 예시: 더미 프로세스로 대상 포트인 3100~3199 대역을 모두 강제 점유한 후 실행 시 스크립트 크래시 발생)
- **환경 변수 파싱 예외 처리 및 로깅 누락**: 기형적인 환경 변수가 주입되어 파싱에 실패하거나 기본값(Fallback)이 적용되는 과정에서, 이를 관리자가 인지할 수 있는 명시적인 경고 로그(`logging.warning` 또는 `logging.error`) 처리가 누락되어 문제 상황 추적이 어렵습니다. (`api/app/core/config.py`)

## Security concerns
- **파일 I/O 시스템 레벨 예외 처리 미흡**: 백엔드 워크플로우 실행 로직에서 워크스페이스 내 파일을 읽고 쓸 때, OS Lock 획득 실패나 권한 에러(PermissionError)에 대한 방어가 불충분합니다. 동시 다발적인 파일 접근 시 프로세스 전체가 크래시되거나 데이터가 손상될 위험이 존재하므로, 안전한 구조화된 예외 처리 및 에러 로깅이 필수적입니다. (`api/app/services/workspace.py`)

## Missing tests / weak test coverage
- **OS Lock 경합 검증 테스트 부재**: 다중 워커 환경에서 파일 접근 시 짧은 순간에 발생하는 OS Lock 경합 상황에 대한 단위 테스트가 부족합니다. `unittest.mock`을 활용해 의도적으로 파일 권한 오류 및 Lock 실패를 유발하고 에러 핸들링을 검증하는 커버리지가 필요합니다. (`api/tests/test_workspace_security.py`)
- **시스템 로깅 통합 검증 누락**: 파손된 `.env` 설정 주입 시 예상대로 로깅 시스템이 동작하는지 검증하는 통합 테스트가 없습니다.
- **UI 엣지 케이스 시각 검증 부재**: 툴팁 DOM에 무작위의 긴 문자열을 주입했을 때 레이아웃이 깨지지 않는지 확인하는 시각적 모의 테스트(Visual Test)가 구성되어 있지 않습니다.

## Edge cases
- **포트 재시도 무한 루프 위험**: 포트 할당 스크립트에 재시도 로직을 추가할 때, 최대 시도 횟수나 명확한 타임아웃 제한이 없으면 사용 가능한 포트(예: 3100번대)가 장기간 해제되지 않을 경우 파이프라인 전체가 무한 대기 상태에 빠질 수 있습니다.
- **순간적인 대량 시스템 에러 발생**: 신규 구현될 시스템 경고 위젯 및 로그 조회 API(`api/app/api/logs.py`)에서 짧은 시간에 대량의 에러가 쏟아질 경우, 조회 쿼리가 병목을 일으키거나 대시보드 위젯에 무리를 줄 수 있으므로 조회 개수 제한(최대 50건)과 성능 방어가 보장되어야 합니다.
- **반응형 툴팁 레이아웃 전환 오류**: 데스크톱의 마우스 레이어 툴팁이 모바일 기기 화면(Bottom Sheet 등)으로 전환될 때, 패딩이나 마진 값 충돌로 인해 텍스트가 잘리거나 화면을 이탈하는 현상이 발생할 수 있습니다.
