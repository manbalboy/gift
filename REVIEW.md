# REVIEW

## Functional bugs
- **API `/resume` 엔드포인트 동시성 이슈**: 다중 클라이언트가 동시에 `/resume` 요청을 보낼 경우 락(Lock) 메커니즘 부재로 인하여 상태 경합이 발생하며 `400 Bad Request` 에러를 반환합니다. 정상적인 동시성 제어를 통해 `200 OK` 또는 `409 Conflict`를 반환하도록 수정이 필요합니다.
- **워크플로우 엔진 예외 처리 누락**: `workflow_engine.py`에서 시스템 자원 접근이나 파일 I/O 작업 중 예외가 발생할 때, 해당 프로세스가 크래시되거나 에러가 무음(Silent) 처리되어 DB의 `Run` 및 `NodeRun` 상태가 `failed`로 커밋되지 않는 버그가 있습니다.
- **UI 상태 반영 불일치**: 노드가 실패(`failed`) 상태로 전이되었음에도 불구하고 대시보드(ReactFlow 캔버스) 상에서 명시적인 붉은색 시각적 렌더링이 반영되지 않으며, 사용자에게 즉각적인 재시도(Retry Node) 액션을 유도하는 기능이 부재합니다.

## Security concerns
- **환경 변수 파싱 취약점**: `DEVFLOW_LOCALHOST_SPOOF_GUARD_PORTS` 등 `config.py` 내 주요 환경 변수 파싱 시 포괄적인 예외 처리가 부족합니다. 극한의 특수문자나 빈 문자열 등 임의의 잘못된 값이 주입될 경우, 애플리케이션 기동이 실패하거나 의도치 않은 동작을 유발할 수 있으므로 안전한 기본값(Fallback) 처리가 필요합니다.
- **로컬 자원 접근 안정성 및 OS Lock 방어 부재**: `workspace.py`에서 파일 읽기/쓰기 수행 시 운영체제 수준의 Lock 획득 경합에 대한 방어 로직이 부족하며, 예외 발생 시 구조화된 에러 로깅이 남지 않아 보안 감사 및 추적에 취약합니다.

## Missing tests / weak test coverage
- **환경 변수 파싱 Unit Test 누락**: `config.py`의 설정 파싱 로직에 대해 빈 문자열이나 특수문자 주입 등 엣지 케이스를 검증하는 단위 테스트가 부족합니다.
- **동시성 및 멱등성 Integration Test 누락**: 다중 클라이언트 환경을 모사하여 `/resume` API를 동시 다발적으로 호출했을 때의 멱등성을 검증하는 통합 테스트가 없습니다. 테스트 시 로컬 서버 포트 `3100` 환경을 활용한 검증이 필요합니다.
- **상태 커밋 및 E2E Test 누락**: 파일 I/O 또는 권한 예외를 고의로 발생시켰을 때 프로세스가 죽지 않고 `failed` 상태로 DB에 기록되는지 확인하는 테스트, 그리고 아티팩트 손상 발생 시 대시보드 노드 다이어그램이 올바르게 에러 상태(붉은색)를 렌더링하는지 확인하는 E2E 테스트가 누락되어 있습니다.

## Edge cases
- **다중 동시 재개 요청 (데드락 위험)**: 50개 이상의 비동기 동시 요청이 `/resume`으로 인입될 경우, DB 트랜잭션과 비동기 락의 범위가 겹쳐 데드락이 발생하거나 무한 대기 상태에 빠질 수 있는 극단적인 상황이 발생할 수 있습니다.
- **파일 권한 부족 및 I/O 간섭**: 워크플로우 실행 중 외부 요인에 의해 작업 디렉터리 권한이 갑자기 변경되거나 아티팩트 파일이 손상된 경우, 오류가 단순히 무시되거나 전체 백그라운드 워커가 크래시되는 상황이 존재합니다. (Graceful degradation 및 실패 커밋 필요)
- **로컬 실행 포트 충돌 상황**: 로컬 환경에서 프리뷰 또는 테스트 서버를 구동할 때 포트가 이미 점유된 상황에 대한 대비가 미흡합니다. 충돌 방지를 위해 `3100`, `3101` 등 3100번대 포트를 동적으로 할당받도록 처리해야 합니다.
