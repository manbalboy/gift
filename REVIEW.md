# REVIEW

## Functional bugs
- `web/src/App.tsx`에서 알림(Toast) 상태를 업데이트하는 함수 내부 로직이 렌더링 사이클과 강하게 결합되어 있어, Ref 데이터(`dedupedToastKeysRef` 등)와 상태(State) 간의 불일치 및 렌더링 충돌을 유발할 수 있습니다.
- `web/src/components/Toast.tsx`에서 창 크기 조절(Resize) 이벤트를 수신할 때 디바운스(Debounce) 또는 쓰로틀(Throttle) 처리가 누락되어 있어, 연속적인 크기 조절 시 과도한 리렌더링과 UI 멈춤 현상이 발생할 수 있습니다.
- 모바일 뷰포트 환경에서 Toast 내용의 텍스트 확장(Expand) 탭을 누를 때, CSS 높이가 갑작스럽게 변동하면서 화면이 덜컹거리는 레이아웃 깨짐(Jumping) 현상이 발생합니다.

## Security concerns
- 외부에서 전달되는 웹훅 페이로드 데이터(예: `error.detail` 등)를 화면에 렌더링할 때, 악의적인 스크립트가 실행될 위험성(XSS)이 존재합니다. 애플리케이션 전체에서 `dangerouslySetInnerHTML` 속성의 사용을 원천적으로 배제하고, React의 기본 텍스트 이스케이프 메커니즘이 모든 외부 주입 데이터에 안전하게 적용되어 있는지 무결성을 검증해야 합니다.

## Missing tests / weak test coverage
- **상태 경합(Race Condition) 단위/통합 테스트 부족**: 앱 내 알림이 최대 개수(3개)에 도달한 시점에서 기존 알림의 만료 이벤트와 동일한 `dedupeKey`를 가진 새로운 웹훅 알림 수신이 밀리초 단위로 겹치는 극한의 상황에 대한 검증 코드가 누락되어 있습니다.
- **모바일 환경 및 UI 상태 분기별 테스트 누락**: `Toast.test.tsx` 내에 모바일 뷰포트 크기를 모킹(Mocking)하여 텍스트 말줄임 처리 로직과 확장 버튼 노출 상태를 검증하는 단위 테스트가 부족합니다.
- **실제 브라우저 환경 반응성 E2E 테스트 부재**: 로컬 환경(예: `http://localhost:3100`)에서 동적인 브라우저 리사이징 및 모바일 화면 터치 이벤트를 발생시켜, 레이아웃의 시각적 결함을 교차 검증하는 Playwright E2E 테스트가 충분하지 않습니다.

## Edge cases
- Toast 컴포넌트가 화면에서 제거(언마운트)될 때, 등록된 `resize` 이벤트 리스너가 완벽하게 클린업되지 않을 경우 메모리 누수(Memory Leak)가 발생하여 장시간 앱 사용 시 성능 저하를 일으킬 수 있습니다.
- 사용자가 확장 탭을 매우 빠르게 연타하거나, 비동기 닫기 타이머가 완료되는 시점에 정확히 새로운 상태 업데이트가 겹치면 앱의 알림 큐가 비정상적으로 꼬이는 상태 동기화 실패 엣지 케이스가 발생할 수 있습니다.
- 모바일 화면에서 여러 알림 카드가 겹쳐(Stack) 있을 때, 특정 카드를 확장하면 밀려나는 다른 카드들의 전환 애니메이션이 어색하거나 화면 영역 밖으로 이탈해버릴 가능성이 있습니다.
