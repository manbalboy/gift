# REVIEW

## Functional bugs
- **DAG Fallback 예외 처리 오류**: `api/app/services/workflow_engine.py` 내에서 연결(엣지)이 없는 독립 노드가 올바르게 병렬 큐로 진입하지 못하고 강제로 순차 실행되는 버그가 있습니다. 
- **웹훅 에러 핸들링 미흡**: 웹훅 수신 시 유효하지 않은 `workflow_id` 또는 잘못된 데이터 포맷이 인입되었을 때, 올바른 HTTP 상태 코드(`422 Unprocessable Entity`)를 즉시 반환하지 못하고 서버 내부 에러나 처리 지연을 유발할 수 있습니다.
- **SSE 커넥션 자원 누수**: 워크플로우 실행 강제 취소 API를 호출하더라도, 기존에 열려있던 비동기 제너레이터(SSE 커넥션)가 즉시 안전하게 해제되지 않아 좀비 커넥션이 남고 메모리 누수로 이어질 가능성이 존재합니다.
- **구동 환경 포트 불일치**: 로컬 환경에서 프론트엔드 및 백엔드 포트가 명확히 분리/고정되지 않아 테스트 시 혼선이 올 수 있습니다. (설계상 UI는 3100번, API는 3101번 포트 고정이 필요합니다.)

## Security concerns
- **Human Gate 권한 검증(RBAC) 누락**: 워크플로우 승인 및 반려를 수행하는 `api/app/api/workflows.py`의 휴먼 게이트 엔드포인트에 인증 토큰만 확인하고, 실제 승인 권한이 있는 사용자(예: `reviewer`, `admin` 역할)인지 식별하는 Role 기반 인가 로직이 부족하여 권한이 없는 사용자가 워크플로우를 통과시킬 위험이 있습니다.
- **감사 로그(Audit Log) 부재**: 휴먼 게이트 단계에서 누가(`decided_by`), 언제(`decided_at`), 어떤 페이로드(`payload`)로 승인/반려를 결정했는지 추적할 수 있는 이력 기록 시스템이 없어 보안 감사 및 SDLC 리뷰 시 취약점이 됩니다.

## Missing tests / weak test coverage
- **Human Gate 라이프사이클 E2E 테스트**: 승인 대기 단계로 진입한 후, 권한 검증 및 실제 화면의 승인/반려 모달 조작을 거쳐 워크플로우가 정상적으로 재개되는지 증명하는 프론트엔드 통합 E2E 테스트(Playwright 등)가 누락되어 있습니다.
- **예외 처리 및 자원 반환 단위 테스트**: 워크플로우 취소 시 `asyncio.CancelledError`가 정상적으로 포착되고 활성화된 SSE 스트리밍이 즉각 종료되는지 검증하는 백엔드 코너 케이스 단위 테스트가 부족합니다.
- **API 유효성 및 인가 테스트**: 잘못된 웹훅 페이로드에 대한 422 상태 코드 응답, 권한이 없는 계정 접근 시의 403 Forbidden 응답 처리 여부를 명확히 증명하는 통합 API 테스트가 없습니다.

## Edge cases
- **인가 실패 시 클라이언트 피드백 부재**: 사용자가 휴먼 게이트 승인을 시도했다가 권한 없음(403) 에러를 맞닥뜨렸을 때, 콘솔에만 에러가 기록되고 화면상에 친절한 폴백 모달이나 안내 문구가 렌더링되지 않아 사용자가 무한정 대기하거나 혼란을 겪을 수 있습니다.
- **네트워크 단절 및 재연결 정책(백오프) 부재**: 사용자의 네트워크 상태가 불안정하여 SSE 연결이 끊어졌을 때, 시스템이 지수적 백오프(Exponential Backoff)를 통해 점진적으로 자동 재연결을 시도하는 복원 로직이 클라이언트 훅(Hook) 단에 존재하지 않습니다.
