# REVIEW

## Functional bugs
- **웹훅 422 예외 처리 누락**: `webhooks.py`에서 `workflow_id`가 유효하지 않거나 페이로드가 잘못된 경우 `422 Unprocessable Entity`를 반환해야 하지만, 현재 적절히 핸들링되지 않아 내부 서버 에러(500) 응답이 발생할 수 있습니다.
- **SSE 커넥션 자원 누수(좀비 연결)**: 클라이언트가 워크플로우 실행을 중간에 취소하거나 네트워크 단절이 발생했을 때, 비동기 제너레이터가 완전히 종료되지 않고 `asyncio.CancelledError`가 정상적으로 처리되지 않아 메모리 누수가 발생할 위험이 있습니다.
- **DAG 병렬 큐 진입 오류**: `workflow_engine.py`의 워크플로우 DAG 순회 로직에서, 이전 노드와 연결(edge)이 없는 독립 노드가 있을 경우 올바르게 병렬 처리 큐에 적재되지 않고 실행이 누락되는 버그가 존재합니다.

## Security concerns
- **Human Gate 권한 인가(RBAC) 부재**: 현재 `workflows.py`의 휴먼 게이트 관련 API(승인/수정/거절) 엔드포인트에 `reviewer` 또는 `admin`과 같은 명확한 Role 기반 인가 로직이 적용되어 있지 않아, 권한이 없는 사용자가 워크플로우 상태를 임의로 조작할 수 있는 보안 취약점이 있습니다.
- **감사 로그(Audit Log) 무결성**: 휴먼 게이트에서 입력된 결정 주체(`decided_by`), 시간(`decided_at`), 페이로드(`payload`) 데이터를 DB에 기록할 때 사용자 입력값에 대한 철저한 검증(Sanitization)이 필요합니다.
- **CORS 및 접근 통제 점검**: 지정된 API 포트(`3101`)에 대해 허용된 origin(로컬 `3100` 포트 및 프리뷰 도메인) 외의 불필요한 크로스 도메인 요청을 차단하고 있는지 점검이 필요합니다.

## Missing tests / weak test coverage
- **프론트엔드 통합(E2E) 테스트 부족**: Playwright를 이용해 프론트엔드 구동 환경(`http://localhost:3100`)에서 휴먼 게이트의 승인/반려 시나리오와 권한 부족 시 403 Fallback 모달이 노출되는 전체 흐름을 검증하는 E2E 테스트 케이스가 불충분합니다.
- **백엔드 자원 해제 테스트 부재**: `asyncio.CancelledError` 발생 시나리오를 Mocking하여 백엔드의 메모리나 네트워크 자원이 적절한 시점에 완전히 반환되는지 확인하는 Pytest 단위 테스트가 보강되어야 합니다.
- **예외 응답 검증 테스트**: 422 오류 및 권한 부족으로 인한 403 상태 코드가 API 계층에서 정확히 반환되는지 확인하는 통합 테스트 스위트가 누락되어 있습니다.

## Edge cases
- **지수적 백오프(Exponential Backoff) 재연결 부하**: 클라이언트(UI 포트 `3100`)에서 SSE 연결 단절 시 자동 재연결을 시도할 때, 서버 장애 상황에서 초기 지연 시간 및 최대 대기 시간이 부적절하게 설정되어 있다면 백엔드(`3101`)에 과도한 트래픽 부하가 집중될 수 있습니다.
- **동시 승인/반려 충돌(Race Condition)**: 휴먼 게이트에서 다수의 리뷰어가 거의 동일한 시점에 승인과 반려를 동시에 요청할 경우, 락(Lock)이나 트랜잭션 제어가 미비하여 상태가 의도치 않게 덮어쓰여지는 경합 조건이 발생할 수 있습니다.
