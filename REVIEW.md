# REVIEW

## Functional bugs
- **초과 루프 카운트 표기 오류**: `LoopMonitorWidget`에서 설정된 최대 루프 횟수를 초과하여 실행될 경우, UI 상에 '0'으로 표기되는 심각한 시각적 버그가 존재합니다. 루프 엔진의 상태를 정확히 반영하지 못하여 사용자가 무한 루프 등의 이상 상태를 인지하기 어렵게 만듭니다. 초과된 횟수가 정확히 표기되고 붉은색 등의 경고 UI가 노출되어야 합니다.
- **SSE 이벤트 렌더링 병목**: 루프 엔진에서 발생하는 잦은 상태 갱신(SSE 이벤트)으로 인해 프론트엔드에서 과도한 리렌더링이 발생하여 브라우저 멈춤 현상(Spike)이 유발될 가능성이 높습니다. 상태 갱신 로직에 디바운싱(Debouncing) 또는 쓰로틀링(Throttling) 적용이 누락되어 있습니다.
- **수동 제어 상태 피드백 지연**: Start, Pause, Stop 등의 엔진 제어 시 백엔드 처리 전까지 UI에 로딩 스피너나 버튼 비활성화 처리가 없어 사용자의 중복 조작을 유발할 수 있습니다.

## Security concerns
- **XSS 필터링 로직 취약점**: `web/src/utils/security.ts`의 `sanitizeAlertText` 함수에서 악성 스크립트 태그 필터링과 정상적인 제네릭 문법(`<T>`) 처리가 충돌하는 보안 결함이 확인되었습니다. DOMPurify의 설정 오류 또는 잘못된 필터링 적용으로 인해 정상 텍스트가 유실되거나 악의적 스크립트 우회 공격에 노출될 위험이 존재합니다.

## Missing tests / weak test coverage
- **대시보드 위젯 단위 테스트 누락**: `web/src/components/LoopMonitorWidget.test.tsx`가 누락되어 있습니다. 색상 톤(`qualityTone`)의 상태별 변화, 수치 렌더링 검증, 루프 횟수 표기 등에 대한 테스트가 시급합니다.
- **에러 상태 통합 테스트 누락**: `web/src/App.test.tsx`에 큐 오버플로우나 루프 중단 등 API에서 `dropped` 상태가 수신될 때 경고 토스트가 정상적으로 렌더링되는지 확인하는 통합 테스트가 부족합니다.
- **보안 엣지 케이스 테스트 부족**: XSS 방어 로직 수정 후 발생할 수 있는 부작용(정상 텍스트 삭제 등)을 검증할 수 있는 단위 테스트 케이스가 보강되어야 합니다.

## Edge cases
- **초고속 상태 변경 시 UI 동기화 문제**: 성능 최적화를 위해 디바운싱/쓰로틀링을 적용했을 때, 사용자가 수동 개입(Pause/Stop)을 시도하는 시점과 UI에 표기된 엔진 상태 간에 미세한 시간차가 발생할 수 있습니다. 이로 인한 상태 불일치(Race condition) 상황을 대비해야 합니다.
- **루프 엔진 장기 실행 중 연결 끊김**: 24시간 장기 실행을 목표로 하나, 네트워크 문제로 SSE 연결이 끊어졌을 때 재연결(Reconnection) 및 누락된 상태 동기화 처리에 대한 명확한 UI/UX 대비책이 고려되어야 합니다.
