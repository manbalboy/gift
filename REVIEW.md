# REVIEW

## Functional bugs
- 잘못된 웹훅 페이로드나 `workflow_id` 전송 시 적절한 HTTP 422 밸리데이션 예외 처리가 누락되어 서버 내부 에러가 발생할 가능성이 있습니다.
- 워크플로우 강제 취소 시 `asyncio.CancelledError`가 정상적으로 핸들링되지 않아 SSE 커넥션 비동기 제너레이터가 종료되지 않고 메모리 누수로 이어지는 결함이 있습니다.
- DAG 워크플로우 순회 로직에서 이전 노드와 연결(edge)이 없는 독립 노드가 누락되어 병렬 처리 큐에 진입하지 못하고 실행을 건너뛰는 버그가 존재합니다.
- 프론트엔드 SSE 커넥션 단절 시 자동 재연결 로직이 불안정하며, 서버 복구 시 클라이언트들이 동시에 연결을 시도하면서 심각한 부하(Thundering Herd)를 유발할 수 있습니다.

## Security concerns
- 휴먼 게이트(승인/수정/거절) API 엔드포인트에 Role 기반 인가 시스템(RBAC)이 없어 권한이 없는 사용자도 임의로 결정 상태를 조작할 수 있는 심각한 취약점이 있습니다.
- 휴먼 게이트의 의사결정에 대한 감사 로그(Audit Log)가 저장되지 않아, 결정 주체, 시간, 페이로드의 추적 및 데이터 무결성 검증이 불가능합니다.
- 프론트엔드에서 휴먼 게이트 접근 시 권한이 없는 사용자(HTTP 403 반환 케이스)를 위한 적절한 Fallback 처리나 안내 UI가 결여되어 있습니다.

## Missing tests / weak test coverage
- 워크플로우 강제 취소에 따른 `asyncio.CancelledError` 핸들링 및 서버 메모리/좀비 프로세스 자원 해제를 검증하는 비동기 단위 테스트가 누락되어 있습니다.
- 잘못된 웹훅 입력 422 반환 예외 검증과 휴먼 게이트의 역할별 403 인가 에러를 확인하는 Pytest 통합 테스트가 부족합니다.
- 프론트엔드 `http://localhost:3100` 환경에서 휴먼 게이트의 승인/반려 절차 및 권한 부족 Fallback 모달의 정상 노출 여부를 자동 검증하는 Playwright 기반 E2E 테스트 스크립트가 없습니다.

## Edge cases
- **Race Condition**: 다수의 리뷰어가 거의 동시에 동일한 휴먼 게이트 승인/반려를 요청할 경우 상태가 의도치 않게 덮어씌워질 수 있습니다. 데이터 무결성 보장을 위한 DB 트랜잭션 락(Lock) 제어가 필수적입니다.
- **Thundering Herd**: 서버 다운 후 복구 시 다수의 클라이언트 재연결이 일시에 몰리는 현상을 방지하기 위해 지수적 백오프(Exponential Backoff) 로직에 Jitter(난수 지연) 처리가 반드시 포함되어야 합니다.
- 워크플로우 그래프 내에 엣지가 하나도 없는 단일 노드만 정의된 극단적인 구성에서도 DAG 순회 로직이 예외 없이 해당 노드를 큐에 적재하도록 처리되어야 합니다.
