# REVIEW

## Functional bugs
- **`resume` API 엔드포인트 멱등성(Idempotency) 결여**: `api/app/api/workflows.py`의 `resume` 엔드포인트에서 다수의 비동기 동시 요청이 유입될 경우 락(Lock) 경합으로 인해 `400 Bad Request` 에러가 응답되는 문제가 있습니다. 상태에 따라 `409 Conflict` 또는 `200 OK`를 일관성 있게 반환하여 멱등성을 보장해야 합니다.
- **워크플로우 실패 상태 전이 처리 미흡**: 아티팩트 유실 등의 예외 상황에서 안전하게 실패 상태로 전이되지 못하고 시스템 예외가 발생합니다. 프로세스 크래시 대신 `Run` 상태 객체를 `failed`로 안전하게 갱신하고 반환해야 합니다.
- **프론트엔드 대시보드 UI 상태 렌더링 불일치**: React 대시보드에서 워크플로우 실패 시 에러 알림(Toast)만 노출되며, 시각적인 노드 다이어그램에는 Failed 상태(붉은색)가 반영되지 않는 시각적 불일치가 존재합니다.

## Security concerns
- **환경 변수 파싱 로직의 취약성**: `config.py`의 `DEVFLOW_LOCALHOST_SPOOF_GUARD_PORTS`와 같은 환경 변수 파싱 시 유효하지 않은 입력이나 특수 문자가 주입되면 파서 크래시가 발생할 수 있습니다. 이는 시스템 초기화 실패나 서비스 거부(DoS)로 이어질 수 있으므로, 안전한 기본값으로 폴백(Fallback)하는 예외 처리가 필요합니다.
- **파일 I/O 및 자원 접근 예외 처리 누락**: 워크스페이스 아티팩트 접근 시 파일 시스템 권한 부족이나 OS Lock에 의한 예외 방어 로직이 미흡합니다. 장기 실행 런타임에서 자원 누수 및 애플리케이션 크래시를 유발할 수 있으므로 포괄적인 예외 처리가 필수적입니다.

## Missing tests / weak test coverage
- **포트 파싱 로직 단위 테스트 부재**: `config.py` 등 설정 파일 내부 파싱 함수에 극한의 입력값(특수기호, 빈 문자열, 비정상적인 길이의 문자열 등)을 주입하여 엣지 케이스를 검증하는 단위 테스트가 누락되어 있습니다.
- **비동기 동시성 통합 테스트 부재**: 수십 개의 클라이언트가 동시에 `resume` API를 호출하는 상황(예: 로컬 포트 `3100`으로 실행된 서버에 부하 인가)에서 락 경합과 멱등성이 올바르게 작동하는지 검증하는 테스트가 필요합니다.
- **DB 상태 커밋 통합 테스트 부재**: Graceful Failure 메커니즘이 작동하여 `Run` 및 `NodeRun` 테이블에 `failed` 상태가 정확히 커밋되는지 확인하는 통합 테스트가 없습니다.
- **UI 실패 상태 E2E 테스트 부재**: 고의로 아티팩트를 손상시킨 뒤 대시보드에 접근했을 때, 노드 다이어그램이 정상적으로 붉은색 실패 상태를 렌더링하는지 확인하는 E2E 테스트 시나리오가 마련되어 있지 않습니다.

## Edge cases
- **다중 스레드/비동기 동시 요청 시 데드락 가능성**: 멱등성 확보를 위해 추가되는 잠금 로직이 기존 트랜잭션과 맞물려 데드락(Deadlock)을 유발하거나 타임아웃을 발생시킬 수 있는 엣지 케이스를 철저히 검토해야 합니다.
- **무음(Silent) 에러 처리의 위험성**: 파일 I/O 오류 처리를 위해 `try-except`를 광범위하게 적용할 경우, 예상치 못한 치명적인 에러까지 덮여 디버깅이 불가능해질 수 있습니다. 반드시 구조화된 로깅과 함께 예외를 캡처해야 합니다.
- **실행 포트 충돌**: 로컬 실행 시 기본 포트가 이미 점유 중일 경우 서비스가 구동되지 않을 수 있습니다. 재현 테스트나 로컬 개발 환경 실행 시에는 임의의 가용한 포트(예: `3100`, `3101` 등)를 할당하여 충돌을 회피하는 엣지 케이스를 고려해야 합니다.
