# REVIEW

## Functional bugs
- **동기식 렌더링에 의한 실행 엔진 구조**: 현재 프론트엔드가 API(GET 조회)를 주기적으로 호출해야만 다음 스텝이 진행되는 클라이언트 의존적인 버그를 내포하고 있습니다.
- **노드 전이 조건 무시**: 노드 간의 실제 연결선(Edges) 로직이 무시되고 단순 배열 순서(Sequence)에 따라 무조건적으로 다음 단계로 넘어가는 구조적인 오류가 확인됩니다.
- **Toast 컴포넌트 렌더링 오류**: 다수의 오류나 알림이 짧은 시간 내에 연속 발생 시 UI가 깨지거나 화면 전체를 덮는 문제가 발생합니다.
- **대용량 아티팩트 렌더링 크래시**: 수십 MB 크기의 로그나 스크린샷 렌더링 시 브라우저에 부하가 걸리며 프리징 현상이 유발됩니다. 

## Security concerns
- **Webhook 무결성 검증 누락**: 외부 연동용 Webhook 수신 엔드포인트(`api/app/api/webhooks.py`)에서 페이로드 위변조를 막는 HMAC 서명 검증 및 IP 대역 차단 기능이 존재하지 않아 인가되지 않은 악의적 호출에 취약합니다.
- **Human Gate 인가 검증 미비**: 워크플로우 진행 중 승인(Approve)/반려(Reject) 처리 시, 요청자의 실제 권한을 검증하는 로직이 없어 누구나 파이프라인 진행 상태를 임의 조작할 수 있는 위험이 존재합니다.
- **자원 회수 실패(Graceful Shutdown 부재)**: 워크플로우 강제 취소 요청 시 백엔드 워커 스레드나 클라이언트로 연결된 SSE 스트림 채널이 안전하게 종료 및 회수되지 않아, 장기 운영 시 메모리 릭과 리소스 고갈의 직접적인 원인이 됩니다.

## Missing tests / weak test coverage
- **웹훅 보안 및 인가 통합 테스트 부재**: IP 위조 및 잘못된 서명(HMAC)이 포함된 외부 요청을 백엔드에서 정상 차단하는지를 검증하는 `pytest` 방어망이 없습니다.
- **워크플로우 재개(Resume) 테스트 누락**: 휴먼 게이트에 도달해 워크플로우가 중단(Interrupt)된 후 다시 재개될 때 컨텍스트가 정상적으로 이행되는지 확인하는 백엔드 통합 테스트가 누락되었습니다.
- **스트림 자원 누수 검증 부족**: 다중 SSE 커넥션 연결 및 강제 종료가 반복되는 시나리오에 대한 스트레스 부하 테스트 스크립트가 없어 엣지 케이스 환경의 시스템 신뢰성이 낮습니다.
- **UI 캔버스 E2E 테스트 부재**: `http://localhost:3100` 대상 환경에서 ReactFlow 에디터 상의 노드 드래그, 순환 참조 연결 검증, 캔버스 저장을 자동화로 점검하는 Playwright E2E 테스트 스크립트가 마련되어 있지 않습니다.

## Edge cases
- **다량의 에러 알림 동시 노출**: 동시에 10개 이상의 오류가 발생하여 프론트엔드로 전달될 경우, 화면 가림을 방지하기 위해 큐를 통해 최대 3개까지만 제한적으로 화면에 노출하고 순차 소거하는 엣지 케이스 처리가 요구됩니다.
- **노드 순환 참조(DAG 위반)**: UI 에디터에서 A노드가 B노드를 가리키고 B노드가 다시 A노드를 가리키는 사이클 연결을 구성할 경우 서버가 크래시되지 않도록 `http://localhost:3100` 프리뷰 저장 및 검증 단계에서 무한 루프 차단 로직이 필수적입니다.
- **포트 충돌에 의한 구동 실패**: 로컬 개발과 E2E 테스트가 동시에 진행될 경우 `http://localhost:3100` 포트가 이미 점유되어 런타임 충돌을 일으키는 상황에 대한 포트 방어 및 안내 예외 처리가 필요합니다.
- **간헐적 네트워크 장애 및 복구 실패**: 외부 API 호출 노드 등에서 일시적인 타임아웃 오류 발생 시 즉시 완전한 실패(Failed)로 처리되지 않고 일정 횟수의 자동 재시도(Backoff Retry)를 수행하여 자가 복구하는 엣지 케이스 로직이 엔진 내에 보강되어야 합니다.
