# REVIEW

## Functional bugs
- **상태 전이 동기화 문제:** `loop_engine.py`에서 Start/Pause/Stop 상태 전이 시 비동기 처리 지연으로 인해 상태가 불일치하는 현상이 발생할 수 있습니다. 특히 장기 실행 중 Inject Instruction이 들어올 때 현재 실행 중인 태스크와의 충돌 방지 로직이 미흡합니다.
- **가상 스크롤 렌더링 지연:** 프론트엔드 대시보드에서 시스템 로그가 급증할 때, UI 컴포넌트에 윈도잉이 제대로 적용되지 않으면 메인 스레드 블로킹 및 프레임 드랍이 발생할 위험이 있습니다.

## Security concerns
- **제어 API 권한 검증 누락 위험:** Self-Improvement Loop 제어 API에 인증/인가 로직이 불완전할 경우, 외부 공격자가 임의로 시스템 루프를 조작하거나 무한 루프를 발생시켜 리소스 고갈을 유발할 수 있습니다.
- **CORS 및 포트 노출 정책:** SPEC에 명시된 CORS 허용 도메인(`manbalboy.com`, `localhost`, `127.0.0.1`) 외의 접근이 철저히 차단되는지 점검해야 합니다. 로컬 테스트를 위해 3100번 포트를 띄울 때 예기치 않은 외부 접속이 허용되지 않도록 주의해야 합니다.

## Missing tests / weak test coverage
- **분산 락 교착 상태(Deadlock) 테스트:** 다중 워커 환경에서 분산 락 획득 후 프로세스가 비정상 종료되었을 때, TTL(Time To Live) 만료에 의해 락이 정상적으로 해제되고 시스템이 복구되는지 검증하는 통합 테스트가 부족합니다.
- **권한 실패 엣지 케이스:** 유효하지 않은 토큰이나 권한이 없는 사용자가 엔진 제어 API를 호출했을 때 정상적으로 차단하고 적절한 HTTP 상태 코드(예: 403 Forbidden)를 반환하는지 확인하는 테스트 케이스가 누락되어 있습니다.

## Edge cases
- **다중 워커 동시 Start 요청:** 여러 관리자나 워커가 동시에 루프 `Start` API를 호출할 경우, 분산 락이 획득되기 직전의 짧은 시간에 시뮬레이터가 중복 실행될 가능성(Race condition)이 존재합니다.
- **대용량 스트리밍 중 네트워크 단절:** 로컬 3100번 포트에서 대용량 로그 스트리밍을 수신하던 중 네트워크가 일시적으로 끊어지고 재연결될 때, 프론트엔드 UI 상에서 로그의 순서가 엉키거나 중복 출력될 수 있습니다.
- **루프 엔진 OOM(Out of Memory):** 장기 실행 중 로그 데이터가 설정된 임계치를 초과하여 지속적으로 유입될 때, 오래된 로그 삭제(Retention) 정책이 제때 동작하지 않으면 백엔드와 브라우저 모두에서 메모리 고갈이 발생할 수 있습니다.
