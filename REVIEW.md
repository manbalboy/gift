# REVIEW

## Functional bugs
- `Toast` 컴포넌트의 상태 초기화 및 렌더링 등 PLAN에 명시된 주요 기능은 정상적으로 동작합니다.
- 단, `Toast.tsx` 내부의 메시지 예외 처리 로직(`const message = typeof item.message === 'string' ? item.message : '';`)으로 인해, `item.message`가 숫자(`123`, `404` 등)로 전달되는 경우 텍스트가 표시되지 않고 빈 문자열로 렌더링되는 기능적 결함(버그)이 발생할 수 있습니다.
- 스와이프 제스처를 시도하다가 임계값(`SWIPE_DISMISS_THRESHOLD`)을 넘지 못해 터치를 뗄 때, 알림이 원래 위치로 돌아가지만 CSS 애니메이션(트랜지션)이 없어 UI가 뚝 끊기며 되돌아가는 사용성 결함이 있습니다.

## Security concerns
- `dangerouslySetInnerHTML` 및 커스텀 이스케이프 함수가 완벽히 제거되었습니다. React의 내장 텍스트 바인딩(`{message}`)을 사용하여 렌더링 구조가 안전하게 변경되었습니다.
- `Toast.test.tsx`의 단위 테스트에서도 `<img src=x onerror=alert(1)>`과 같은 페이로드가 스크립트로 실행되지 않고 문자열 그대로 노출됨을 증명하고 있어, XSS 방어 등 보안 요구사항을 완벽히 충족합니다.

## Missing tests / weak test coverage
- **[P1] 전역 알림 대기열(Queue) 단위 테스트**: 최대 3개 노출 제한, 초과 시 대기열 큐잉 동작, 그리고 고유 키(`dedupeKey`) 기반 중복 필터링 테스트가 `App.test.tsx` 내에 충실하게 작성되었습니다.
- E2E 환경(`toast-layering.spec.ts`)에서 모바일 스와이프, 뷰포트 레이어 유지, 4개 이상 닫기 등의 테스트가 원활히 커버되었습니다. 로컬 실행 시 요구된 3100번대 포트 환경에서도 문제없이 수행할 수 있는 구조입니다.
- **부족한 부분**: 사용자가 알림의 긴 텍스트를 읽기 위해 마우스를 올리고(Hover) 있을 때 자동 만료 타이머(`durationMs`)를 일시 정지(Pause)하는 기능 및 해당 테스트가 누락되어 있습니다. 또한, 키보드 탐색(Tab 키) 시 접근성에 대한 테스트가 다소 부족합니다.

## Edge cases
- **반응형 뷰포트 제스처 방어**: 모바일 환경에서 스와이프 도중 창을 리사이즈해 데스크톱으로 전환할 때(`isMobile` 변경 시), 잔상이 남지 않도록 스와이프 오프셋을 `0`으로 초기화하는 클린업 엣지 케이스 방어가 훌륭하게 구현되었습니다.
- **오버플로우 렌더링 방어**: `clientWidth <= 10` 조건 및 `document.fonts.ready` 지연 측정 로직을 통해 렌더링 타이밍 차이로 발생하는 오버플로우 오작동 엣지 케이스를 성공적으로 차단했습니다.
- **멀티 터치 방어**: 두 손가락 이상으로 터치(`event.touches.length > 1`)할 경우 스와이프로 인식되지 않게 조치하여 예기치 않은 모바일 제스처 오작동을 방어했습니다.
