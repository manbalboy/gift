# REVIEW

## Functional bugs
- `web/scripts/check-port.mjs` 실행 시 동시성(Race Condition) 문제가 있습니다. 다중 인스턴스/워커 환경에서 포트를 할당할 때 점유 유예 기간 및 넉넉한 재시도 대기 시간이 부족하여, 여러 워커가 동일한 포트를 획득하려다 충돌하는 현상이 발생할 수 있습니다.
- 백엔드 시스템 로그 API(`/api/logs`)에서 최근 50건의 로그를 역순으로 조회할 때, 데이터베이스 스키마 내 `created_at` 컬럼에 대한 인덱스가 적용되지 않아 로그가 누적될수록 전체 테이블 스캔으로 인한 심각한 성능 저하가 일어납니다.

## Security concerns
- 시스템 경고 및 에러 알림 컴포넌트(`SystemAlertWidget`)를 통해 프론트엔드로 전달되는 로그 페이로드에 민감 정보(내부 서버 경로, 인프라 환경변수, 인증 토큰 일부 등)가 마스킹 없이 그대로 렌더링 될 위험이 존재합니다.
- Preview 외부 노출 환경 구성 시 허용된 도메인(manbalboy.com 및 localhost 계열) 이외의 접근 시도에 대해 CORS 및 외부 포트(7000-7099) 접속 차단 정책이 정상적으로 작동하는지 보안 재점검이 필요합니다.

## Missing tests / weak test coverage
- 프론트엔드 알림 UI(`SystemAlertWidget` 및 에러 툴팁)에 대한 Playwright 기반 E2E 시각적 테스트가 누락되어 있습니다. 특히 데스크톱 및 모바일 뷰포트별 렌더링 호환성 검증이 부재합니다.
- 인프라 포트 할당 스크립트가 타임아웃 처리 로직을 가지고 있는지 입증할 자동화 쉘 스크립트 테스트가 부족합니다.
- 백엔드 인덱스 최적화 작업 후, 실제로 쿼리 스캔 효율이 개선되었음을 확인하는 API 성능/단위 테스트 커버리지가 부족합니다.

## Edge cases
- 에러 메시지에 띄어쓰기가 없는 긴 문자열이나 비정상적인 텍스트 노드가 포함될 경우, 툴팁 내부에서 `word-break` 처리가 되지 않아 부모 컨테이너(Bounding Box)의 가로 레이아웃을 이탈하고 UI가 붕괴될 가능성이 있습니다.
- 런타임 중 여러 건의 경고 알림이 동시에 발생하여 스택이 쌓일 때, 알림 위젯의 전체 세로 크기가 뷰포트의 `max-height`를 초과하여 화면을 벗어나고 스크롤조차 되지 않는 현상이 발생할 수 있습니다.
- 포트 스캔 로직 구동 시 인프라 내 3100번대 포트(예: 3100~3199) 전체가 다른 프로세스에 의해 이미 선점되어 여유 포트가 하나도 없는 극한의 상황에서 스크립트가 타임아웃 종료되지 않고 무한 대기할 우려가 있습니다.
