# REVIEW

## Functional bugs
- **프론트엔드 폼 개행 누적 버그**: 대시보드의 Reject Reason 및 지시 주입 폼에서 프리셋 기능(`handleRejectReasonPreset`) 클릭 시 개행 문자 및 후행 공백이 지속적으로 누적되는 버그가 있습니다. 입력 폼의 문자열 처리 정규식을 최적화하여 텍스트 누적 시 발생하는 불필요한 여백 생성을 수정해야 합니다. (`web/src/App.tsx`)
- **빌더 UI 사전 유효성 검사 누락**: Workflow Builder UI에서 단절된 그래프(Disconnected graph)나 다중 진입점을 가진 비정상적인 워크플로우 상태를 구성했을 때, 클라이언트 캔버스 단에서 이를 사전에 차단하지 못하고 백엔드로 저장 요청이 전송되는 문제가 있습니다. 

## Security concerns
- **로컬 미들웨어 뷰어 토큰 인증 취약점**: 프록시를 우회하여 로컬 포트(예: `http://localhost:3100`)를 통해 API 서버에 직접 접근할 경우 보안 취약점이 발생할 수 있습니다. 전역 API 라우팅 단계에서 뷰어 토큰(Viewer Token)의 유효성을 엄격하게 검증하는 방어 코드가 부재하여 권한 우회 가능성이 존재합니다. (`api/app/api/dependencies.py` 등)

## Missing tests / weak test coverage
- **프론트엔드 단위 테스트 누락**: 폼 프리셋 텍스트 입력 시 발생하는 개행 및 공백 처리 최적화 로직에 대한 단위 테스트가 누락되어 있습니다. (`web/src/App.test.tsx`)
- **E2E 테스트 검증 부족**: 단절된 워크플로우 그래프나 다중 진입점이 존재하는 상태에서 사용자가 저장을 시도할 때, UI 상에서 에러 피드백(토스트 알림 등)이 정상적으로 렌더링되는지 확인하는 통합 테스트 시나리오가 부족합니다. (`web/tests/e2e/WorkflowBuilder.spec.ts`)
- **백엔드 그래프 검증 테스트 한계**: 백엔드의 워크플로우 검증 로직 테스트가 아직 과거 사양에 머물러 있어 단절 그래프를 허용하는 방식으로 작성되어 있습니다. 비정상적인 그래프 데이터 접수 시 백엔드에서 `400` 또는 `422` 유효성 에러를 올바르게 반환하는지 확인하도록 테스트 코드를 전면 개편해야 합니다. (`api/tests/test_workflow_engine.py`)

## Edge cases
- **Agentic Loop (무한 루프) 예산 초과**: 24시간 실행되는 초장기 실행(Autopilot) 환경의 특성상, 특정 AI 노드가 예상치 못한 사유로 무한 반복 루프에 빠질 경우 시스템 리소스(비용/시간 예산)를 급격히 고갈시킬 위험이 있습니다. 엔진 실행 라이프사이클 전반에 걸쳐 노드 실행 시 최대 반복 횟수를 카운팅하고, 임계치 초과 시 엔진 상태를 강제로 일시 정지(Pause) 시켜 사람의 개입을 기다리거나 실행을 중단하는 방어 로직이 필요합니다.
