# REVIEW

## Functional bugs
- **포트 할당 스크립트의 레이스 컨디션**: `web/scripts/check-port.mjs` 실행 시 다중 워커가 동시에 동일한 로컬 포트(예: 3100)를 점유하려고 시도할 때 경합 조건(Race Condition)이 발생하여 워크플로우 엔진이 예기치 않게 중단될 수 있습니다. 포트 할당 시 점유 유예 시간 및 재시도 간격에 대한 조정이 필요합니다.
- **알림 위젯 UI 세로 오버플로우**: 프론트엔드의 `SystemAlertWidget` 컨테이너에 긴 텍스트를 가진 에러 로그가 연속으로 쌓일 경우, 세로 방향으로 오버플로우가 발생하여 UI가 뷰포트를 이탈하거나 전체 레이아웃이 깨지는 현상이 있습니다. 컴포넌트에 대한 `overflow-y: auto` 및 `max-height` 설정이 누락되어 있습니다.
- **로그 조회 쿼리 성능 저하**: 대시보드에서 백엔드 시스템 알림 로그(`/api/logs`)를 조회할 때, `created_at` 컬럼 기준의 역순 정렬 스캔에 대한 데이터베이스 인덱스가 구축되어 있지 않아 로그 데이터 누적 시 심각한 응답 지연이 발생할 수 있습니다.

## Security concerns
- **시스템 알림 로그 내 민감 정보 노출 (Log Sanitization 누락)**: 백엔드 에러 발생 시 생성되는 알림 페이로드나 프론트엔드 렌더링 화면에 시스템 절대 경로(예: `/home/docker/`, `/root/`) 및 인증 토큰(`Bearer ...`) 등 인프라의 민감한 정보가 마스킹 처리 없이 그대로 노출될 위험이 존재합니다. 정규표현식을 활용한 치환 유틸리티 적용이 필수적입니다.
- **외부 지시 주입 및 제어 API 접근 관리**: 워크플로우 엔진 v2의 실행 상태를 제어(중단/재개)하고 지시를 주입하는 기능이 도입되면서, 악의적인 접근이나 권한이 없는 사용자의 API 호출을 방어하기 위한 최소한의 인가 검증 체계 점검이 요구됩니다.

## Missing tests / weak test coverage
- **UI 및 반응형 시각적 E2E 테스트 부재**: `SystemAlertWidget`의 긴 단어 줄바꿈(word-break) 및 모바일 뷰포트 레이아웃 안정성을 검증하는 Playwright 기반의 시각적 E2E 테스트(`web/tests/e2e/system-alert.spec.ts`)가 누락되어 있습니다.
- **포트 고갈 상황 시뮬레이터 및 예외 테스트 누락**: 로컬 실행 환경에서 가용한 포트(예: 3100, 3101 등 3100번대 포트)가 모두 소진되었을 때를 가정한 BASH 기반 타임아웃 시뮬레이터(`web/scripts/test-port-timeout.sh`) 구축 및 이를 검증하는 테스트 커버리지가 비어 있습니다.
- **예산 차단(Budget Limit) 단위 테스트 부족**: 에이전틱 워크플로우가 무한 루프에 빠지지 않도록 재시도 횟수나 시간 예산을 초과했을 때, 오케스트레이터가 노드 실행을 정상적으로 차단하고 블로킹(Blocked) 처리하는지 확인하는 백엔드 단위 테스트 작성이 필요합니다.

## Edge cases
- **극단적 로컬 포트 고갈 현상**: 다수의 동시 작업이 발생하여 가용한 모든 3100번대 로컬 포트가 소진될 경우, 프로세스가 무한 대기 상태(Deadlock)에 빠지지 않고 명시적으로 타임아웃을 발생시킨 뒤 안전하게 실패를 반환하는지 엣지 케이스 검증이 필요합니다.
- **LLM 무한 루프에 의한 비용 폭주**: 계획 수립이나 코드 구현 노드에서 LLM 에이전트가 스스로 작성한 결과물에 갇혀 반복적으로 동일한 실패 로직을 생성할 때, 시스템이 이를 선제적으로 감지하여 중단(Cancel) 상태로 강제 전환하는 방어 로직이 요구됩니다.
