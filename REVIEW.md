# REVIEW

## Functional bugs
- 리버스 프록시(예: Nginx) 환경에서 프론트엔드(3100 포트)와 백엔드(3101 포트) 간의 SSE 연결이 장기간 유지될 때, Keep-Alive 주기 설정이 적절하지 않아 타임아웃 및 연결 끊김이 발생할 가능성이 높습니다.
- 대시보드의 Audit Log 탭에서 대량의 `status.md` 파일을 실시간으로 로드하고 파싱하는 경우 성능 병목 및 UI 렌더링 지연이 우려됩니다. 아티팩트 데이터를 효율적으로 추출하고 캐싱하는 로직이 필요합니다.
- 프론트엔드 측의 SSE 연결이 일시적으로 끊어졌을 때를 대비한 자동 재연결 로직과 재연결 시 누락된 이벤트를 동기화하는 기능이 불충분하여 상태 불일치를 유발할 수 있습니다.

## Security concerns
- `status.md` 또는 아티팩트의 텍스트 내용을 프론트엔드 대시보드에 렌더링할 때, 잠재적인 악성 스크립트 삽입에 의한 XSS(Cross-Site Scripting) 공격 위험이 존재합니다. 내용 출력 전 안전한 살균(Sanitization) 처리가 필수적입니다.
- 백엔드(3101 포트) API 호출 및 웹훅 수신 시 CORS 설정 오류로 인한 비인가 접근 위험이 있습니다. `SPEC.md`에 명시된 허용 origin(`manbalboy.com`, `localhost`, `127.0.0.1`)만을 정확하게 허용하도록 엄격하게 구성해야 합니다.

## Missing tests / weak test coverage
- Nginx 리버스 프록시가 포함된 모의 도커 환경에서 네트워크 강제 지연 및 유휴 상태를 유발하는 SSE 스트리밍 부하/복원력 테스트(Resilience test)가 누락되어 있습니다.
- 백엔드 스케줄러가 장기 미처리 휴먼 게이트(24시간 초과)를 정확히 식별하는지 검증하는 타임 모킹(Time Mocking) 단위 테스트 커버리지가 부족합니다.
- Playwright 기반의 E2E 테스트에서 Audit Log 필터링, 날짜 지정 쿼리 및 결과 렌더링에 이르는 UI 상호작용 자동화 시나리오가 추가되어야 합니다.

## Edge cases
- 사용자가 네트워크 지연 등의 이유로 휴먼 게이트의 승인(Approve), 거절(Reject) 버튼을 연타하여 동일한 상태 변경 요청이 동시에 백엔드로 인입되는 경우 멱등성이 깨져 상태 충돌이 일어날 수 있습니다.
- Audit Log 검색 기능 사용 시 서버와 클라이언트 간의 타임존(Timezone) 불일치로 인하여 특정 시간 경계에 있는 이벤트나 아티팩트가 검색 결과에서 제외되는 현상이 발생할 수 있습니다.
- 워크플로우 런이 실패하여 폴백 플로우(`default_linear_v1`)로 전환되는 과정 중, 이미 생성된 휴먼 게이트 요청이 닫히지 않고 고아(Orphan) 상태로 무한 대기할 수 있는 엣지 케이스가 존재합니다.
